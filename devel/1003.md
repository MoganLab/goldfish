## 任务相关的代码文件
- `tools/fmt/liii/goldfmt.scm` - `gf fmt --exclude PATTERN` 参数解析、目录与变更文件遍历时按 basename 跳过被排除文件

## YYYY/MM/DD 为 fmt / fix 增加 --exclude

### What
为 `gf fmt` 和 `gf fix` 新增 `--exclude` 参数，递归遍历目录或处理 `--changed-since` 文件列表时跳过匹配的文件：

```bash
bin/gf fmt --exclude=prefix-kbd.scm /path/to/dir
bin/gf fix --exclude=prefix-kbd.scm /path/to/dir
bin/gf fmt --exclude=a.scm,b.scm /path/to/dir
bin/gf fmt --changed-since=HEAD --exclude=prefix-kbd.scm
```

行为边界如下：

- `--exclude PATTERN` 按 basename 比较，因此 PATTERN 既可传 basename（`prefix-kbd.scm`），也可传完整相对路径（`TeXmacs/progs/texmacs/keyboard/prefix-kbd.scm`）。
- 多个 PATTERN 用逗号分隔，空串被丢弃。
- 排除判定同时识别 `/` 与 `\` 路径分隔符，跨平台一致。
- `gf fix` 内部会对目标先调一次 `gf fmt`；这次调用同样透传 `--exclude`，否则被排除文件仍会在 fmt 阶段被改动。
- `--exclude` 对单文件路径（`gf fmt file.scm`）不生效——既然用户显式指定了文件，就不再跳过。

### Why
此前 `gf fmt` / `gf fix` 递归遍历目录时会无差别处理所有 `.scm`，没有跳过特定文件的能力。下游项目（如 Mogan）存在一些文件受人工维护风格约束，不应被自动格式化，否则会产生与维护意图冲突的大段 diff。

没有 `--exclude` 时，下游只能在项目侧自行收集文件清单、逐文件调用 `gf fmt`，既慢（每文件重启一次解释器）又复杂（要处理路径分隔符、缓存、计数等）。`--exclude` 把跳过逻辑下沉到二进制侧，下游只需传一个参数即可。

### How
#### 1. 新增公共 helper（goldfmt / goldfix 各自内联，避免新增共享文件）

- `(path-basename path-str)`：取路径末段，同时识别 `/` 与 `\`，兼容 Windows 路径。
- `(file-excluded? entry-str excludes)`：取 entry 与每个 pattern 的 basename 比较，命中返回 `#t`。
- `(parse-excludes raw)`：拆分逗号分隔值，丢弃空串；`raw` 为 `#f` 或空串时返回 `'()`。

#### 2. `gf fmt` 接入 --exclude

`make-fmt-arg-parser` 新增：

```scheme
(parser :add-argument '((name . "exclude") (type . string)))
```

主入口读取并解析：

```scheme
(excludes (parse-excludes (parser 'exclude)))
```

`excludes` 透传给三个遍历入口：

- `format-directory`：在 `path-file?` 分支，extension 匹配后再判 `(not (file-excluded? entry-str excludes))`，递归子目录时也带 `excludes`。
- `format-file-list`：处理文件列表前先判 exclude，命中则跳过、不计入 total。
- `format-changed-since`：把 `excludes` 透传给 `format-file-list`。

单文件路径分支不应用 exclude（用户显式指定的文件不跳过）。

#### 3. `gf fix` 接入 --exclude

结构与 fmt 一致。额外一点：`fix-directory` 和 `fix-file` 内部会先 `os-call gf fmt <path>`，所以这次 fmt 调用也拼上 `--exclude`：

```scheme
(let ((exclude-arg (if (null? excludes) ""
                      (string-append " --exclude=" (string-join excludes ",")))))
  (os-call (string-append (executable) " fmt" exclude-arg " " dir-path)))
```

`string-join` 来自 `(srfi srfi-13)`，已在 import 列表中。

#### 4. 帮助文档

`gf fmt --help` / `gf fix --help` 各加一行选项说明与一条 Example。

### Notes
- 排除按 basename 比较，这意味着同名文件在所有目录下都会被跳过。当前下游需求（排除特定文件）下这是可接受的；若未来需要按完整路径精确排除，可扩展匹配规则。
- goldfish 现有测试均为子模块纯函数级（scan/format/repair 等），没有 CLI/目录遍历的端到端 fixture。本任务不新增单测，依赖发版后手动 smoke test 验证。
