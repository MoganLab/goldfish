## 2026/05/05 接口设计

### What
设计 `(srfi srfi-215)` 和 `(liii logging)` 的日志模块接口。

### Why
- SRFI-215 定义了 Scheme 的中央日志交换标准接口
- `(liii logging)` 提供与 SRFI-215 兼容但更友好的实现
- 最终用户采用 `(liii logging)` 而非直接使用 `(srfi srfi-215)`

### How

#### (srfi srfi-215) 接口定义

`(srfi srfi-215)` 严格按照 SRFI-215 标准完整实现：

```scheme
(define-library (srfi srfi-215)
  (export
    send-log
    current-log-fields
    current-log-callback
    EMERGENCY ALERT CRITICAL ERROR WARNING NOTICE INFO DEBUG)
  (import (scheme base)
          (scheme write))
  (begin
    ;; 完整实现：包括日志级别常量、plist 字段处理、
    ;; current-log-fields/current-log-callback 参数、send-log 函数
  ))
```

导出符号：

| 符号 | 类型 | 说明 |
|---|---|---|
| `send-log` | 函数 | `(send-log severity message . plist)` 发送日志消息，plist 风格如 `'(FACILITY 1)` |
| `current-log-fields` | 参数 | 全局默认字段，值为 **plist**，如 `'(FACILITY 1 USER "admin")` |
| `current-log-callback` | 参数 | 日志回调函数，接收 alist 形式的日志条目 |
| `EMERGENCY` | 常量 | 严重级别 0，系统不可用 |
| `ALERT` | 常量 | 严重级别 1，必须立即采取行动 |
| `CRITICAL` | 常量 | 严重级别 2，临界条件 |
| `ERROR` | 常量 | 严重级别 3，错误条件 |
| `WARNING` | 常量 | 严重级别 4，警告条件 |
| `NOTICE` | 常量 | 严重级别 5，正常但重要的情况 |
| `INFO` | 常量 | 严重级别 6，信息性消息 |
| `DEBUG` | 常量 | 严重级别 7，调试级别消息 |

#### (liii logging) 接口设计

`(liii logging)` 不依赖 `(srfi srfi-215)`，但接口与 SRFI-215 保持一致，并额外提供便捷函数。

**SRFI-215 兼容层：**

| 符号 | 类型 | 说明 |
|---|---|---|
| `send-log` | 函数 | `(send-log severity message . plist)`，与 SRFI-215 完全一致 |
| `current-log-fields` | 函数 | 读取全局默认字段（返回 plist） |
| `current-log-callback` | 函数 | 读取当前日志回调函数 |
| `EMERGENCY` ~ `DEBUG` | 常量 | 与 SRFI-215 相同的严重级别常量 |

**设置函数：**

| 符号 | 说明 |
|---|---|
| `(log-set-fields! plist)` | 设置全局默认字段，plist 风格如 `'(FACILITY 1 USER "admin")` |
| `(log-set-callback! callback)` | 设置日志回调函数 |
| `(log-set-file-handler! path)` | 设置文件日志处理器，内部调用 `(make-file-handler path)` |
| `(current-log-fields-in-alist alist)` | 用 alist 风格设置字段，如 `'((FACILITY . 1) (USER . "admin"))` |

**便捷函数（按级别）：**

| 符号 | 说明 |
|---|---|
| `(log-emergency message . args)` | 发送 EMERGENCY 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-alert message . args)` | 发送 ALERT 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-critical message . args)` | 发送 CRITICAL 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-error message . args)` | 发送 ERROR 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-warning message . args)` | 发送 WARNING 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-notice message . args)` | 发送 NOTICE 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-info message . args)` | 发送 INFO 级别日志，支持 `pyfmt` 具名格式化 |
| `(log-debug message . args)` | 发送 DEBUG 级别日志，支持 `pyfmt` 具名格式化 |

格式化规则：`message` 是 `pyfmt` 格式字符串，`args` 为 plist 参数。`pyfmt` 格式化后的结果即为日志消息内容（对应 `%(message)s`）。当 `args` 为空时，`message` 直接作为日志内容。

**日志级别控制：**

| 符号 | 类型 | 说明 |
|---|---|---|
| `log-set-level!` | 函数 | `(log-set-level! severity)` 设置最小输出级别，低于此级别的日志被静默丢弃 |
| `current-log-level` | 函数 | 返回当前设置的最小日志级别 |
| `log-set-format!` | 函数 | `(log-set-format! format-string)` 设置日志输出格式字符串，如 `"%(asctime)s [%(levelname)s] %(message)s"` |
| `current-log-format` | 函数 | 返回当前设置的日志格式字符串 |
| `log-flush!` | 函数 | 强制刷新当前日志输出缓冲 |

**格式化函数（由 `(liii string)` 提供）：**

| 符号 | 说明 |
|---|---|
| `(pyfmt format-string . plist)` | Python `%` 风格的具名格式化，将 `%(key)s` 替换为 `plist` 中对应字段值。`plist` 的键既可以是关键字（如 `:name`），也可以是符号（如 `'name`） |

**默认行为：**

当 `current-log-callback` 未设置时，日志按级别自动分流：
- `ERROR` 及以上级别（`EMERGENCY`、`ALERT`、`CRITICAL`、`ERROR`）输出到 `current-error-port`（stderr）
- 低于 `ERROR` 的级别（`WARNING`、`NOTICE`、`INFO`、`DEBUG`）输出到 `current-output-port`（stdout）

**默认处理器：**

| 符号 | 说明 |
|---|---|
| `(default-log-handler message port)` | 默认日志格式化输出到指定端口，使用 `current-log-format` 格式字符串。默认格式：`"%(asctime)s [%(levelname)s] %(message)s"` |
| `(make-stdout-handler)` | 创建输出到 `current-output-port` 的日志回调函数，无缓冲，立即输出 |
| `(make-stderr-handler)` | 创建输出到 `current-error-port` 的日志回调函数，无缓冲，立即输出 |
| `(make-file-handler path)` | 创建输出到指定文件路径的日志回调函数。追加模式，内部带缓冲（满 10 条或 4KB 时自动 flush），自动创建父目录，自动换行 |

**辅助函数：**

| 符号 | 说明 |
|---|---|
| `(log-message-severity msg)` | 从日志条目中获取 SEVERITY |
| `(log-message-message msg)` | 从日志条目中获取 MESSAGE |
| `(log-message-field msg key)` | 从日志条目中获取指定字段值 |

#### 日志条目格式

日志条目是 alist，至少包含：
- `(SEVERITY . severity)` - 严重级别（0-7 的整数）
- `(MESSAGE . message)` - 日志消息字符串

额外字段通过 `send-log` 的 plist 参数或 `current-log-fields` 传递。
`(liii logging)` 额外提供 `current-log-fields-in-alist`，方便用 alist 风格设置字段。

`current-log-format` 支持以下预定义占位符（通过 `pyfmt` 替换）：
- `%(asctime)s` - 当前时间，格式 `YYYY-MM-DD HH:MM:SS.SSS`
- `%(levelname)s` - 级别名称，如 `INFO`、`DEBUG`、`ERROR`
- `%(severity)d` - 级别数字，如 `6`、`7`、`3`
- `%(message)s` - 日志消息内容

字段值类型：
- `string` - 字符串
- `exact integer` - 精确整数
- `bytevector` - 字节向量
- `error-object` - 错误对象
- 其他类型自动通过 `write` 转为字符串

#### 使用示例

```scheme
(import (liii logging))

;; 设置最小日志级别为 INFO
(log-set-level! INFO)

;; 设置默认输出到 stderr
(log-set-callback! (make-stderr-handler))

;; 发送日志
(log-info "服务启动完成")
(log-debug "调试信息")  ; 被过滤，不输出
(log-error "发生错误" 'CODE 500)

;; 使用 send-log（SRFI-215 兼容）
(send-log WARNING "磁盘空间不足" 'DISK "/var")

;; 输出到文件
(current-log-callback (make-file-handler "/var/log/app.log"))

;; 使用 Python 风格的具名格式化
(pyfmt "User %(name)s logged in" :name "Bob")
;; => "User Bob logged in"

(pyfmt "User %(name)s is %(age)d years old" 'name "Alice" 'age 30)
;; => "User Alice is 30 years old"
```
