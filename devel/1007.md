## 3. gf fmt 用法

```bash
gf fmt                       # 仓库批量格式化所有语言（读 gf_fmt.json：C++ + Scheme）
gf fmt --check               # CI 非破坏检查，未格式化则退出码 1
gf fmt file.scm              # 格式化单个文件（按后缀选语言）
gf fmt --dry-run file.scm    # 预览格式化结果（不写回）
gf fmt dir/                  # 递归格式化目录下所有 .scm 文件
gf fmt -e scm,sld dir/       # 指定后缀（逗号分隔多个）
gf fmt --changed-since=HEAD  # 仅格式化自 HEAD 以来变更的 Scheme 文件
gf fmt --exclude=PATTERN ... # 追加排除（逗号分隔多个）
```

入口分发顺序（`cond` 自上而下短路）：
- `--changed-since` 优先：用户传了该 flag 时（无论是否带路径）一律走 Scheme 增量。
- 无路径参数：读 `gf_fmt.json`，`--check` 走非破坏检查，否则仓库批量；配置缺失则报错退出 1。
- 有路径参数：单文件按后缀选语言（cpp 后缀拒绝并提示用 clang-format，其余走 scheme-fmt）；目录递归。

## 4. gf_fmt.json 格式

放项目根（`g_project-root`，即 `gfproject.json` 所在目录）：

```json
{
  "cpp": {
    "suffix": "hpp,cpp,h,c,cc,cxx",
    "path": "src",
    "exclude": [
      {"path": "src/s7*", "reason": "s7 体系文件，沿用 s7 自身代码风格"},
      "tests"
    ]
  },
  "scheme": {
    "suffix": "scm",
    "path": "goldfish,tools",
    "exclude": ["tests"]
  }
}
```

- `suffix`：逗号分隔的后缀（不带点）；未写则用语言默认（cpp=`hpp,cpp,h,c,cc,cxx`，scheme=`scm`）。
- `path`：逗号分隔的根目录，仓库批量/check 时递归收集。
- `exclude`：**每语言各自**。同时支持两种格式——纯字符串 `"src/s7*"` 或对象 `{"path":"...","reason":"..."}`。支持通配符 `*`。

排除匹配规则（`goldfmt-lang.scm` 的 `path-matches-exclude?`）：
- pattern 含 `*`：glob 匹配。含 `/` 时对完整路径匹配（`src/s7*` 精确到 src 下），否则对 basename 匹配（`test*` 匹配任意目录下 test 开头）。
- pattern 不含 `*`：精确路径匹配（等于 pattern，或以 `/pattern` 结尾，使 `tests` 同时匹配 `tools/fmt/tests` 与 `tests`）。

## 5. 如何测试

### 5.1 确定性测试
```bash
xmake b goldfish

# 仓库批量 + CI 检查：仓库已全部格式化时应输出 OK、退出码 0
bin/gf fmt --check; echo "exit=$?"

# 向后兼容：单文件 / 目录递归
bin/gf fmt tools/fmt/liii/scheme-fmt.scm
bin/gf fmt goldfish/liii/
```

### 5.2 非确定性测试（行为验证）
```bash
# 人为制造一处未格式化的 Scheme 文件，确认 --check 能报告并退出码 1
cp tools/version/liii/goldversion.scm /tmp/gv.bak
printf '\n\n   (display "broken")\n' >> tools/version/liii/goldversion.scm
bin/gf fmt --check; echo "exit=$?"   # 期望 exit=1，并列出该文件
cp /tmp/gv.bak tools/version/liii/goldversion.scm   # 还原

# 排除验证：src/s7* 被 cpp.exclude 排除，制造异常不应被报告
printf '\n\n\nint x;\n' >> src/s7.h
bin/gf fmt --check; echo "exit=$?"   # 期望 exit=0（s7.h 不在检查范围）
git checkout src/s7.h                 # 还原

# 配置缺失：无路径参数时报错退出（单文件模式不依赖配置，仍可工作）
mv gf_fmt.json /tmp/
bin/gf fmt; echo "exit=$?"                       # 期望 error: 找不到 gf_fmt.json，exit=1
bin/gf fmt goldfish/liii/base.scm; echo "exit=$?" # 期望正常工作
mv /tmp/gf_fmt.json .
```

## 7. 设计要点

### 7.1 What
1. `gf fmt` 一个命令覆盖全场景：单文件、目录递归、仓库批量、CI 非破坏检查（`--check`）、增量（`--changed-since`）。
2. 单一配置文件 `gf_fmt.json` 按语言（`cpp` / `scheme`）组织，承载扫描范围（`path`）与排除（`exclude`）。
3. 按语言拆分实现：`goldfmt-config`（配置）、`goldfmt-lang`（语言抽象 + 通用工具）、`scheme-fmt`、`cpp-fmt`，主入口 `goldfmt.scm` 仅做参数解析与分发。新增语言只需加一个 `xxx-fmt.scm` 并在 `gf_fmt.json` 增一段。
4. Debian 13 格式化检查 CI：构建 `bin/gf` 后运行 `gf fmt --check`。

### 7.2 Why
goldfish 已有 `gf fmt`（Scheme）和 `.clang-format`（C++），需要**统一入口**和 **CI 级格式化门禁**：C++ 此前无自动化格式化/检查，Scheme 只能逐目录手动跑。把仓库批量与 CI 检查合并进 `gf fmt`、配置集中到 `gf_fmt.json`，使命令职责清晰、配置不再割裂、跨语言可扩展。

扫描范围配置化而非硬编码，是为了让**同一个 `gf fmt` 在不同仓库复用**：下游仓库只需放一份自己的 `gf_fmt.json` 即可。

排除上需要特殊对待两类文件：
- **s7 体系文件**（`src/s7*`）：clang-format 会破坏其脆弱的宏定义与排版（实测导致 `use_write_t`/`shared_info_t`/`sigjmp_buf` 等编译错误），必须整体排除。
- **测试文件**（`tests`）：含大量故意畸形的夹具（缺括号等，交给 fmt 会抛 read-error），且改动频繁，纳入仓库级 format 既慢又易误报；测试格式化由各测试自身负责。

### 7.3 How
- **语言抽象**：`(liii goldfmt-lang)` 维护语言→handler 注册表（`register-lang!`）；各语言实现一组过程（`collect`/`format-files`/`check-file`）并在模块加载时自注册。仓库批量/check 在主入口按 `(lang-names)` 遍历分发。
- **退出码**：s7 tool 的 `main` 返回值由 C++（`goldfish_finish_tool_success`）转为进程退出码；`(exit N)` 也会真正终止进程并传码。故 check 失败、配置缺失、单文件 cpp 拒绝等路径都用 `(exit N)` 传码。
- **exclude 复用**：`glob-match?` / `path-matches-exclude?` / `file-excluded?` / `collect-files` 作为通用层放在 `goldfmt-lang.scm`，scheme-fmt 与 cpp-fmt 共用。
- **check 实现**：Scheme 逐文件 `scan-file`+`format-nodes` 与磁盘逐字节比；C++ 逐文件 `clang-format --dry-run --Werror` 看退出码（Windows 无 sh 跳过，CI 在 Debian 跑）。格式化模式：C++ 用 `clang-format -i --files=<list>` 批量，Scheme 走 goldfmt 引擎 + 文件缓存。
- **不重写算法**：Scheme 格式化核心（goldfmt-scan/format）、文件缓存、clang-format 定位与批量调用均沿用既有实现。

## 8. 演进历史

- **初版（已废弃）**：曾以独立的 `gf format` 子命令（`tools/format/liii/goldformat*.scm`）做仓库级统一格式化，扫描范围用 `gfformat.json`（`cpp-roots`/`scm-dirs`）、排除用 `gfexclude.json`，C++ 走 clang-format、Scheme 调 `gf fmt`。
- **本次重构**：删除 `gf format` 及其三个实现文件、删除 `gfformat.json` 与 `gfexclude.json`，能力合并进 `gf fmt`；配置三合一为 `gf_fmt.json`，按语言拆分 `scheme-fmt` / `cpp-fmt` 模块。
- **gf fix 的排除配置**：`gf fix` 原读 `gfexclude.json`，由另一 PR 改读 `gf_fmt.json` 的 `scheme.exclude`。本 PR 直接删除 `gfexclude.json`，fix 在另一 PR 合并前静默降级（文件不存在返回空排除），不崩溃。
