## 2026/05/05 扩展 ascii-upcase/ascii-downcase 支持字符串，修复 codepoint->hexstr

### What
1. 扩展 `(srfi srfi-175)` 中的 `ascii-upcase` 和 `ascii-downcase`，使其支持字符串参数
2. `ascii-string?` 改为使用 `string-length` + `string-ref` 直接遍历（替代原来的 `call-with-port` + `open-input-string`）
3. 字符串参数必须是纯 ASCII 字符串，否则抛出 `value-error`
4. `goldfish/liii/unicode.scm` 中的 `codepoint->hexstr` 将 `string-upcase` 替换为 `ascii-upcase`
5. 更新四个测试文件的文档，使用"相关函数"一节显式引用 `string-upcase` ↔ `ascii-upcase`、`string-downcase` ↔ `ascii-downcase`

### Why
- `string-upcase`/`string-downcase` 已改为仅由 `(scheme char)` 提供，`(liii unicode)` 不应依赖它
- `codepoint->hexstr` 只需要转换 ASCII 范围内的十六进制字符（0-9, a-f），使用 `ascii-upcase` 更合适
- `ascii-upcase`/`ascii-downcase` 的字符串版本对纯 ASCII 处理场景很有用

## 2026/05/05 移除 C 层面实现，基于 char-*case 重新实现 string-*case

### What
1. 从 `src/s7_liii_string.c` 和 `src/s7_liii_string.h` 中移除 `g_string_upcase`、`g_string_downcase` 的 C 实现
2. 从 `src/s7.c` 中移除 `string-upcase`、`string-downcase` 的符号注册、`set_function_chooser` 及 `main_names` 列表中的引用
3. 移除 `goldfish/scheme/char.scm` 中对 S7 C 内置 `string-upcase`、`string-downcase` 的依赖（删除 `s7-string-upcase`、`s7-string-downcase` 绑定）
4. 在 `goldfish/scheme/char.scm` 中导入 `(liii unicode)`，利用其 UTF-8 编解码函数实现字符串遍历
5. 新增内部辅助函数，按 UTF-8 字符粒度遍历字符串，对每个完整字符调用 `char-upcase` / `char-downcase` / `char-foldcase`
6. 重新实现 `string-upcase`、`string-downcase`、`string-foldcase`
7. 从 `(srfi srfi-13)` 和 `(liii string)` 中移除 `string-upcase`、`string-downcase` 的导出，确保只有 `(scheme char)` 提供这两个函数
8. 删除 `tests/liii/string/string-downcase-test.scm` 和 `tests/liii/string/string-upcase-test.scm`（功能由 `tests/scheme/char/` 覆盖）
9. 更新 `tests/liii/json/json-reduce-test.scm` 导入 `(scheme char)` 以使用 `string-upcase`
10. 补充 UTF-8 测试用例（中文、拉丁扩展字符等）

### Why
- S7 C 内置的 `string-upcase` / `string-downcase` 基于单字节 `toupper` / `tolower`，不支持 UTF-8 多字节字符
- 已有的 `char-upcase`、`char-downcase`、`char-foldcase` 已在 Scheme 层实现了 Unicode simple casing，支持多字节字符
- 字符串版本应当与字符版本行为一致，都支持 Unicode

### How
1. **字符串遍历**：S7 的 `string-ref` 按字节索引，无法直接提取多字节 UTF-8 字符。因此借助 `(liii unicode)`：
   - `string->utf8` 将字符串转为字节向量
   - `bytevector-advance-utf8` 计算下一个 UTF-8 字符的字节边界
   - `utf8->codepoint` + `integer->char` 提取完整字符
   - 调用 `char-upcase` / `char-downcase` / `char-foldcase` 转换
   - `char->integer` + `codepoint->utf8` 转回字节向量
   - 拼接后用 `utf8->string` 得到结果
2. **移除 C 实现**：从 `src/s7_liii_string.c`/`src/s7_liii_string.h` 中删除 `g_string_upcase`、`g_string_downcase`，从 `src/s7.c` 中移除符号注册和 `main_names` 引用
3. **统一入口**：从 `(srfi srfi-13)` 和 `(liii string)` 中移除 `string-upcase`、`string-downcase` 的导出和内部定义，确保只有 `(scheme char)` 提供这两个函数
4. **测试补充**：在现有 ASCII 测试基础上，增加 UTF-8 字符（如中文 `中`、拉丁扩展 `é` 等）的转换验证
