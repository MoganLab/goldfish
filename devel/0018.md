## 2026/05/05 修改 gf test 的默认行为

### What

修改 `bin/gf test` 在无参数时的默认行为，使其根据当前 git 状态智能路由：

1. 如果当前是 git 仓库且分支不是 `main` → 执行 `bin/gf test --changed-since=main`
2. 其他情况（是 `main` 分支，或不是 git 仓库）→ 执行 `bin/gf test tests`

新增 `--all` 参数，采用贪心策略：
- 如果是 `main` 分支 → 运行全部测试（与默认行为一致）
- 如果是其他分支 → 先运行 `--changed-since=main`，再运行全部测试

GitHub Actions 中全部采用 `bin/gf test --all`。

输出格式要求：
- 第一行：说明路由到实际命令的逻辑，并提示用户使用 `gf test --help` 查看细节
- 第二行：显示实际执行的命令（使用 gf 的绝对路径）

### Why

`bin/gf test` 运行所有测试用例需要的时间太久了。开发者在特性分支上通常只关心与本次修改相关的测试。通过智能路由，可以减少开发者在特性分支上的测试等待时间，同时保证 main 分支和 CI 中仍运行全量回归测试。

### How

1. 在 `goldtest.scm` 中新增函数判断当前 git 状态：
   - 使用 `g_os-call` 执行 `git rev-parse --abbrev-ref HEAD` 获取当前分支名
   - 通过命令退出码判断是否在 git 仓库中
2. 新增 `--all` 参数解析函数 `parse-test-all`。
3. 修改 `run-goldtest` 函数：
   - 当 `parse-test-args` 返回无参数时，检查 git 状态进行路由
   - 非 `main` 分支且无 `--all` → 只运行 `changed-since=main`
   - 非 `main` 分支且有 `--all` → 先运行 `changed-since=main`，再运行全部（changed 测试排在前面）
   - `main` 分支或非 git 仓库 → 运行全部测试
   - 按照要求格式打印路由信息和实际命令（使用 `executable` 获取 gf 绝对路径）
4. 保持原有显式参数的逻辑不变，仅在无位置参数时触发智能路由。
