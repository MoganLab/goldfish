## 2026/05/03 重新实现 char-foldcase 和 string-foldcase
### What
按照 Unicode Simple Case Folding 规范，在 `(scheme char)` 的 Scheme 层重新实现 `char-foldcase`，使其行为与 `char-downcase` 区分，并对齐 Chez Scheme 的 Unicode simple casing 行为。同时更新 `string-foldcase` 以基于新的 `char-foldcase` 实现。

1. 使用 Chez Scheme 10.3.0 遍历所有有效 Unicode 码点，收集 `char-foldcase` 和 `char-downcase` 的映射结果
2. 对比两者差异，收集 `char-foldcase` 与 `char-downcase` 行为不同的字符
3. 在 `goldfish/scheme/char.scm` 中实现 `char-foldcase`：先处理与 `char-downcase` 的差异字符（主要是 Cherokee 字母范围和少量希腊/拉丁变体），其余调用 `char-downcase`
4. 更新 `string-foldcase`，使其基于 `char-foldcase` 逐字符映射，而非直接调用 `string-downcase`
5. 更新测试用例，覆盖 ASCII、Cherokee、希腊字母变体等边界情况

### Why
- 当前 `char-foldcase` 直接调用 `char-downcase`，在 Unicode 下两者行为并不等价
- 例如 Cherokee 大写字母（U+13A0–U+13F5）在 `char-foldcase` 下应保持不变，但 `char-downcase` 会映射为 Cherokee 小写
- `string-foldcase` 同样直接调用 `string-downcase`，存在相同问题

### How
1. **数据提取**：使用 Chez Scheme 10.3.0 遍历所有有效 Unicode 码点（跳过代理区 `#xD800`–`#xDFFF`），收集 `char-foldcase` 和 `char-downcase` 的映射结果。
   ```scheme
   ;; 提取 char-foldcase 映射
   (let loop ((cp 0))
     (when (< cp #x110000)
       (unless (and (>= cp #xD800) (< cp #xE000))
         (let* ((ch (integer->char cp))
                (fc (char-foldcase ch))
                (fc-cp (char->integer fc)))
           (when (not (= cp fc-cp))
             (format #t "(~a ~a)~%" cp fc-cp))))
       (loop (+ cp 1))))

   ;; 对比 char-foldcase 和 char-downcase，找出差异
   (let loop ((cp 0))
     (when (< cp #x110000)
       (unless (and (>= cp #xD800) (< cp #xE000))
         (let* ((ch (integer->char cp))
                (fc (char-foldcase ch))
                (dc (char-downcase ch)))
           (when (not (char=? fc dc))
             (format #t "cp ~x foldcase ~x downcase ~x~%"
                     cp (char->integer fc) (char->integer dc)))))
       (loop (+ cp 1))))
   ```
2. **范围合并**：差异字符约 198 个，但大部分是连续范围：
   - Cherokee 大写（U+13A0–U+13F5）在 foldcase 下保持不变
   - Cherokee 小写扩展（U+13F8–U+13FD）foldcase 映射到 Cherokee 大写
   - Cherokee 小写（U+AB70–U+ABBF）foldcase 映射到 Cherokee 大写
   - 少量希腊字母变体、拉丁字母和历史字符需要单独处理
3. **Scheme 实现**：`char-foldcase` 使用 `cond` 表达式先覆盖所有与 `char-downcase` 的差异字符和范围，最后 `else` 分支调用 `char-downcase`。
4. **string-foldcase 更新**：使用 `string-map` 基于 `char-foldcase` 逐字符转换。
   ```scheme
   (define (string-foldcase str)
     (unless (string? str)
       (error 'type-error "string-foldcase: parameter must be string"))
     (string-map char-foldcase str))
   ```
5. **更新测试**：增加 Cherokee 字母、希腊字母变体（如 U+03D0–U+03D6、U+03F0–U+03F5）等测试用例。

### 已知问题
`bin/gf fmt` 会破坏 Unicode 字符字面量。因此 `tests/scheme/char/char-foldcase-test.scm` 不应运行 `bin/gf fmt`。
