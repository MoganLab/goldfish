## 5. 2026/06/23 用 `gf fmt` 格式化三个文件

### 5.1 What
对以下三个文件运行 `bin/gf fmt`，仅做格式（缩进、换行、尾注释）调整，不改语义：

1. `goldfish/guenchi/json.scm`：`write-json` 的 vector 分支里 `when (> i 0)` 体折行并补 `;when`；pair 分支 `let loop` 参数表折到下一行、`when (> i 0)` 同样折行；`cond` 各分支的尾注释 `;vector?` / `;pair?` 归一为 `;`。
2. `goldfish/liii/subprocess.scm`：
   - `os-call` 分支里 `(pair? cmd-spec)` 一行写法的嵌套 `let-values` 折成单行；
   - `%valid-stdout?` / `%valid-stderr?` 的多参数 `or` 改为每参一行、收尾 `;or`；
   - 两个 `unless` 里的 `value-error` + `format` 长字符串折行；
   - `run` 内部 `(apply run-values command (append ...))` 调用重排缩进，尾注释从 `;run-values` 改为 `;apply`（实际收尾的是 `apply`）。
3. `goldfish/liii/sys.scm`：`which` 的 `if path ...` 分支折成单行。

### 5.2 Why
这三个文件此前存在手动写就的不统一缩进与超长行（如 `value-error` 的长 format 串、`%valid-stdout?` 的单行 `or`），与项目 `gf fmt` 的既定风格不一致。统一格式化，便于后续 review 与 `gf fmt` 在仓库内的幂等收敛。

### 5.3 How
直接对三个文件依次运行 `bin/gf fmt <file>`，原地修改后用 `git diff` 人工核对：确认所有改动都是格式层（缩进 / 折行 / 尾注释标签）的变化，没有任何语义改动（无变量重命名、无逻辑分支变更、无表达式增删）。
