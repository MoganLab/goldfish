## 5. 2026-06-15 修复 Linux CI 因缺失 ninja 导致的构建失败

### 5.1 What
修复 Debian、Fedora 以及 XPack Build 三个 CI 工作流在 GitHub Actions 上失败的问题。

1. 在 `ci-debian.yml` 的 `apt-get install` 步骤中新增 `ninja-build`。
2. 在 `ci-fedora.yml` 的 `dnf install` 步骤中新增 `ninja-build`。
3. 在 `package.yml` 的 `apt-get install` 步骤中新增 `ninja-build`。

### 5.2 Why
xmake 的 `package.tools.cmake` 在构建 `cpr` 等从源码编译的第三方包时，默认使用 Ninja generator。GitHub-hosted 的 macOS/Ubuntu runner 预装了 ninja，所以 macOS CI 一直正常；但项目使用的 `debian:trixie` 和 `fedora:43` 容器只装了基础编译工具链，没有 ninja，导致 CMake 配置阶段直接报错：

```text
CMake Error: CMake was unable to find a build program corresponding to "Ninja".
```

### 5.3 How
在三个受影响的 CI 工作流的依赖安装步骤中统一加上 `ninja-build`，保持与 macOS CI 环境一致。这是最小改动，也符合 xmake 生态的常规做法。
