## 2026/06/12

### What
新增 `path-write-bytes` 函数，支持将 bytevector 以二进制模式写入文件。

### Why
在 Windows 上，Scheme 的 `open-output-file` 以文本模式打开文件，写入二进制数据时 `\n` 会被转换为 `\r\n`，导致 PDF、PNG 等二进制文件损坏。macOS/Linux 无此问题。

Mogan 的 HTML 导出功能在 Windows 上因此受影响：写入临时 PDF 文件后 MuPDF 无法正确渲染，最终导出的 HTML 图片为空白。

### How
在 `src/liii_path.cpp` 中新增 `f_path_write_bytes` 函数：
- 使用 `tb_file_init` + `TB_FILE_MODE_WO | TB_FILE_MODE_CREAT | TB_FILE_MODE_TRUNC` 以二进制模式打开文件
- 接收 s7 bytevector，通过 `s7_byte_vector_elements` 和 `s7_vector_length` 获取数据和长度
- 使用 `tb_file_writ` 直接写入，避免任何文本模式转换
- 加文件锁保证写入原子性

在 `goldfish/liii/path.scm` 中新增 `path-write-bytes` 导出，检查参数类型为 bytevector 后调用 `g_path-write-bytes`。
