## 2026/05/07 实现 (liii config-parser) 库

### What
1. 创建 `goldfish/liii/config-parser.scm`，参考 Python configparser 模块实现 INI 配置文件解析库
2. 提供 16 个核心 API 函数：
   - 读取：`config-read-string`、`config-read-file`
   - 查询：`config-sections`、`config-has-section?`、`config-has-option?`、`config-get`、`config-get-int`、`config-get-boolean`、`config-options`、`config-items`
   - 修改：`config-set!`、`config-add-section!`、`config-remove-section!`、`config-remove-option!`
   - 输出：`config-write`
3. 创建 `tests/liii/config-parser-test.scm`，包含 28 个测试场景、64 个断言
4. 在 `README.md` 和 `README_ZH.md` 的标准库表格中加入 `(liii config-parser)` 条目

### Why
参考 Python 的 configparser 模块实现更完整的 INI 配置文件解析库。

### How
**数据结构：**
- 使用 `define-record-type` 创建 `<config-parser>` 记录类型
- 内部用 hash-table 存储各 section 的键值对
- 使用 `string-comparator`（基于 SRFI-128）确保字符串 key 的正确比较

**核心设计：**
- DEFAULT section 继承：查询 option 时先查当前 section，未找到再查 DEFAULT
- Key 大小写不敏感：存储时一律用 `string-downcase` 转为小写
- 分隔符同时支持 `=` 和 `:`，注释同时支持 `;` 和 `#`
- Boolean 识别：支持 yes/no、true/false、on/off、1/0
- `hash-table-for-each` 参数顺序为 `(proc ht)` 而非 `(ht proc)`
