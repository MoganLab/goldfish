# [1004] `(liii path)` Windows 路径解析重构与 pathlib 对齐

## 任务目标
重构 `(liii path)` 的路径解析,修 Windows 平台 bug,并逐步对齐 Python `pathlib` 语义。

## 任务相关的代码文件
- `goldfish/liii/path.scm` - 核心实现
- `tools/fmt/liii/goldfmt-rule.scm` - `path-has-common-ancestor?` 用 `path-parts` + 本地过滤
- `tests/liii/path/` - path 库测试(49 个用例文件)
- `src/liii_path.cpp` - C 层 fs 原语(只读,未改;决定 Scheme 层能补哪些功能)

## pathlib 对照清单

图例:🟢 = 本分支之前已有(部分本轮修复/重构);🔵 = 本分支新增;⚠️ = 简化版,与 pathlib 有差异。

### PurePath(纯路径操作)

| pathlib | path.scm | 状态 |
|---|---|---|
| `parts` | `path-parts` | 🟢 posix 绝对含 `/` stub |
| `name` / `stem` / `suffix` | `path-name` / `path-stem` / `path-suffix` | 🟢(stem/suffix 抽 `split-name-dots`) |
| `suffixes` | `path-suffixes` | 🔵 |
| `parent` | `path-parent` | 🟢 重构为 record parts,去尾斜杠 |
| `parents` | `path-parents` | 🔵 |
| `drive` / `root` | `path-drive` / record `root` | 🟢(无独立 anchor getter) |
| `is_absolute()` | `path-absolute?` | 🟢 Windows 要求 drive+root |
| `is_relative()` | `path-relative?` | 🟢 |
| `joinpath()` / `/` | `path-join` | 🟢 修绝对参数重置 |
| `relative_to()` | `path-relative-to` | 🔵 anchor 不匹配报错 |
| `with_name/stem/suffix` | `path-with-name/stem/suffix` | 🔵 |
| `match()` | `path-match` | 🔵 glob 末段匹配(`*`/`?`/`[seq]`) |
| `as_posix()` | `path-as-posix` | 🔵 |
| `__eq__` | `path-equals?` / `path=?` | 🟢 |

### Path(文件系统)

| pathlib | path.scm | 状态 |
|---|---|---|
| `cwd()` / `home()` | `path-cwd` / `path-home` | 🟢 home 加 USERPROFILE 回退 |
| `exists/is_file/is_dir` | `path-exists?/file?/dir?` | 🟢 |
| `read_text/bytes` | `path-read-text/bytes` | 🟢 |
| `write_text/bytes` | `path-write-text/bytes` | 🟢 |
| `touch/unlink/rename/rmdir` | 同名 | 🟢 unlink 支持 missing-ok |
| `iterdir()` | `path-list-path` | 🟢 |
| `copy()` | `path-copy` / `path-copy-into` | 🟢 |
| `mkdir()` | `path-mkdir` | 🔵 parents / exist-ok |
| `absolute()` | `path-absolute` | 🔵 不解析 symlink |
| `expanduser()` | `path-expanduser` | 🔵 `~`(不支持 `~user`) |
| `resolve()` | `path-resolve` | 🔵⚠️ 仅规范化 `.`/`..`,不解析 symlink(无 realpath 原语) |

## 提交记录(最新在上)

### 2026/06/17 继续对齐 pathlib:补 6 个 API、清理手搓循环
**新增:** `path-mkdir`(parents/exist-ok)、`path-absolute`、`path-expanduser`(`~`)、`path-match`(glob 引擎 `glob-match?`+`charclass-match-one`)、`path-as-posix`、`path-resolve`(简化版,无 symlink 解析)。
**清理:** `path-from-parts` drive 分支手写过滤 loop 改用 `vector-filter`。
**踩坑:** s7 里 `pi` 是预定义常量(π),不可作参数名 → 改 `p0`/`s0`。

### 2026/06/17 代码审查:修 3 个 bug、补 7 个 API
**Bug:** path-parent 返回带尾斜杠(改基于 record parts);path-join 双斜杠+不重置绝对参数(`/a`+`/b`→`/a//b`,实现 pathlib 重置语义);path-home Windows 缺 HOMEDRIVE 崩溃(加 USERPROFILE 回退)。
**新增:** `path-parents`、`path-suffixes`、`path-with-name/stem/suffix`、`path-relative-to`。
**附带:** `drop-dot-parts` 过滤空段(`/tmp/`→`/tmp`);`path-stem`/`path-suffix` 抽 `split-name-dots`。

### 2026/06/17 修复 goldfmt-rule 共同祖先判定回归
重构后 `path-parts` 的 posix 绝对 stub 从 `""` 变 `"/"`,`goldfmt-rule` 的 `string-null?` 过滤失效 → `segments-list` 的过滤改为同时认 `/`、`\`、空串为锚点(本地 `anchor-or-empty?`),不引入额外公共 API。

### 2026/06/16 重写 Windows 路径解析,对齐 pathlib 语义
**任务起因:** s7 `(port-filename)` 在 Windows 返回正斜杠绝对路径(`C:/...`),下游 Mogan autosave 用 `(path-parent (port-filename))` 把脚本目录加入 `*load-path*`,原 path-parent 只认 `\`、整串当单 part → `(import (liii autosave))` 崩溃。
**实现:** `<path>` record 加 `root` 字段区分 `C:\foo`(drive-absolute)与 `C:foo`(drive-relative);重写 `parse-windows-path` 为 `parse-unc` + 主分发,覆盖 UNC / drive-absolute / drive-relative / current-drive-root 四种形式,对齐 `pathlib.PureWindowsPath`;Windows 上同时识别 `/` 和 `\`;大量手写 loop 改用 `(liii string)`/`(liii vector)`。
