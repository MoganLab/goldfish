## 2026/04/07

### What

1. 将流式 POST 能力并入 `http-post`
   - 通过 `:stream #t` 启用按 chunk 处理 POST 响应体
   - 接口形式为：
     `http-post url [params] [data] [headers] [proxy] [files] [output-file] [stream] [callback]`

2. 在 `http-post` 中增加两种流式响应处理方式
   - `:stream #t` + `:callback`：按 chunk 处理响应体
   - `:stream #t` + `:output-file`：按 chunk 将响应体写入本地文件

3. 保持 `http-post` 原有语义不变
   - 不传 `:stream` 时仍返回统一的 response-object
   - `:files` 仍然用于 multipart 文件上传

4. 移除单独的 `http-stream-post`
   - 不再保留独立的流式 POST 主入口
   - 对外统一通过 `http-post` 使用 POST 能力

### Why

原来的 POST 能力被拆成了两个主入口：

- `http-post` 负责普通 POST 和文件上传
- `http-stream-post` 负责按 chunk 处理 POST 响应

这种拆分和现在 `http-get` 已经统一到 `:stream #t` 的设计不一致，也会让用户在使用 POST 时多记一个额外函数。

把流式 POST 收回到 `http-post` 里，可以让 GET 和 POST 的接口风格保持一致，也能减少重复的公开 API。

### How

1. 在 `goldfish/liii/http.scm` 中扩展 `http-post`
   - 新增 `:output-file`、`:stream`、`:callback`
   - 在 `:stream #t` 时要求至少提供 `:output-file` 或 `:callback`
   - 普通 body、multipart `:files` 和流式响应处理都统一经过 `http-post`

2. 在 `src/goldfish.hpp` 中统一 POST 入口
   - `g_http-post` 同时承担普通 POST 和流式 POST
   - 当 callback 为过程时，底层通过 `WriteCallback` 按 chunk 回调响应体
   - 原来的 `g_http-stream-post` 绑定被删除

3. 调整测试与索引
   - 将流式 POST 用例并入 `tests/liii/http/http-post-test.scm`
   - 删除单独的 `tests/liii/http/http-stream-post-test.scm`
   - 更新 `tests/liii/http-test.scm` 与函数索引

4. 通过在线端点验证功能
   - 使用 `httpbin.org/post` 验证普通 POST 不回退
   - 验证 `:stream #t + :callback` 能按 chunk 处理响应
   - 验证 `:stream #t + :output-file` 能将响应落盘
   - 验证 `:files + :stream #t` 能同时完成文件上传与流式响应处理
