## 5. 2026-05-31 从 s7.c 拆分基础函数和谓词函数

### 5.1 What

将 `s7.c` 中的基础函数和谓词函数按功能类别拆分至多个独立的 C 文件，降低 `s7.c` 的复杂度并提高可维护性。

1. **新建/扩展的文件**
   - `src/s7_scheme_base.c`：基础算术运算、let 操作、字符串辅助、列表操作等
   - `src/s7_scheme_predicate.c`：类型谓词（`type?` 系列）、等价性判断（`eq?`/`eqv?`/`equal?`）、比较函数等
   - `src/s7_scheme_write.c`：端口相关函数（`current-input-port`/`current-output-port`/`current-error-port`/`open-output-string` 等）
   - `src/s7_ctables.c`：字符分类表初始化
   - `src/s7_dtoa.c`：Grisu2 双精度浮点转字符串算法
   - `src/s7_op_names.c`：操作码名称表 `op_names[]`
   - `src/s7_internal_helpers.h`：类型谓词共享宏和内部辅助宏
   - `src/s7_liii_vector.c`：复数向量/浮点向量/整数向量的 ref/set 操作

2. **迁移的主要函数类别**
   - **算术基础函数**：`g_add_4`、`g_subtract_3`、`g_quotient`、`g_remainder`、`g_modulo`、`g_less_2`、`g_num_eq_2`、`g_leq_2`、`g_geq_2` 等
   - **let/rootlet 相关**：`g_curlet_ref`、`g_outlet`、`g_rootlet`、`g_curlet`、`g_unlet_disabled`、`g_outlet_unlet`、`g_unlet_ref`、`g_sv_unlet_ref`
   - **列表操作**：`g_memq_2`/`g_memq_4`、`g_memv`、`g_assq`/`g_assv`、`g_list_0`~`g_list_4`、`g_append_2`、`g_reverse`
   - **类型谓词**（约 40+ 个）：`not`、`boolean?`、`symbol?`、`gensym?`、`syntax?`、`let?`、`null?`、`pair?`、`list?`、`number?`、`integer?`、`real?`、`rational?`、`complex?`、`char?`、`string?`、`vector?`、`byte-vector?`、`hash-table?`、`iterator?`、`continuation?`、`random-state?`、`undefined?`、`eof-object?`、`byte?`、`float?`、`macro?`、`input-port?`、`output-port?`、`c-pointer?`、`openlet?`、`funclet?`、`c-object?`、`constant?`、`goto?`、`port-closed?`、`equal?`、`equivalent?`、`eq?`、`eqv?`、`tree-cyclic?`、`type-of` 等
   - **其他谓词/访问器**：`arity`、`help`、`numerator`、`denominator`、`iterator-sequence`、`c-pointer-info`、`c-pointer-type`、`c-object-type`、`c-object-let`、`format-nr`、`tree-leaves`、`cyclic-sequences`、`object-to-let`、`pair-line-number`、`port-line-number`、`tree-memq`、`tree-set-memq`、`tree-set-memq-syms` 等
   - **向量操作**：`g_cv_ref_2`、`g_cv_set_3`、`g_fv_ref_2`、`g_iv_ref_2`

### 5.2 Why

`s7.c` 作为 Goldfish Scheme 的核心解释器实现，体积庞大（超过万行），函数职责混杂，导致：
- 编译时间长，任何小改动都触发全量重编译
- 代码定位困难，类型谓词、算术运算、端口 I/O 等各类函数交织在一起
- 不利于团队协作和后续模块化扩展

通过按功能拆分，可将 `s7.c` 的代码量减少约 1500 行，各模块职责清晰，未来新增 Scheme 内置函数可直接在对应子文件中实现。

### 5.3 How

**整体策略**：以函数为单位，按"功能类别"整体迁移，每次迁移一个或一组紧密相关的函数，确保每一步编译通过、测试通过。

**具体步骤**：
1. **识别函数归属**：根据函数语义判断其所属类别（谓词、算术、I/O、字符串等）
2. **创建目标文件**：如果目标模块文件尚不存在则新建，否则追加
3. **迁移实现**：将函数体从 `s7.c` 剪切到目标 `.c` 文件
4. **声明头文件**：在对应的 `.h` 文件中添加 `extern` 声明
5. **添加桥接（必要时）**：如果目标文件需要访问 `s7.c` 中的内部符号，新增 `s7i_` 前缀的 bridge 函数暴露必要接口
6. **更新构建**：在 `xmake.lua` 中注册新文件
7. **运行测试**：确保 `bin/gf test --changed-since=main` 通过

**谓词函数的统一模式**：
大量类型谓词（如 `symbol?`、`number?` 等）具有相同的结构：
```c
static s7_pointer g_is_xxx(s7_scheme *sc, s7_pointer args) {
  return(s7_make_boolean(sc, xxx(s7_car(args))));
}
```
通过提取共享宏到 `s7_internal_helpers.h`，后续新增谓词只需一行定义即可。

**多次重构与迁移调整**：
- 部分函数在迁移过程中曾先进入 `s7_scheme_predicate.c`，后根据语义被重新归类到 `s7_scheme_base.c`（如算术快捷函数、`assq`/`memv`/`append`/`list` 系列、`rootlet`/`curlet` 等）
- 最终形成稳定的模块边界：`s7_scheme_base.c` 负责核心运算和数据结构操作，`s7_scheme_predicate.c` 负责类型判断和等价性比较
