## 2026/03/22 新增 (liii option) 模块

### What
新增轻量级的 `(liii option)` 模块，提供函数式编程中的 Option 类型支持。

1. 定义 `none` 和 `option` 构造函数
2. 实现 Option 类型的内部表示：
   - `(cons #f 'N)` 表示 `none`
   - `(cons value 'S)` 表示 `(option value)`（some）
3. 提供基本的 Option 操作方法：
   - `none` - 创建空的 option
   - `option` - 创建包含值的 option
   - `option?` - 判断是否为 option 类型
   - `option-map` - 映射函数
   - `option-filter` - 过滤函数
   - `option-flat-map` - 扁平映射函数
   - `option-for-each` - 副作用遍历函数
   - `option-get` - 获取值（为空时报错）
   - `option-get-or-else` - 获取值或默认值
   - `option=?` - 比较两个 option 是否相等
   - `option-or-else` - 提供备选 option
   - `option-defined?` - 判断是否包含值
   - `option-every` - 全称量词检查
   - `option-any` - 存在量词检查
   - `option-empty?` - 判断是否为空
4. 创建对应的测试文件

### Why
提供一种轻量级的 Option 类型实现，用于处理可能缺失的值，避免空值检查带来的代码复杂度。

### How
使用简单的 cons 对作为内部表示：
- `'N` 和 `'S` 标记符号用于区分 none 和 some 两种状态
- 这种表示方式简洁高效，便于模式匹配和类型检查
