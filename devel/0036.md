## 2026/05/05 将 char-ci 和 string-ci 函数改为纯 Scheme 实现

### What
1. 在 `goldfish/scheme/char.scm` 中重写 char-ci 和 string-ci 函数：
   - `char-ci=?`, `char-ci<?`, `char-ci>?`, `char-ci<=?`, `char-ci>=?` 改为基于 `char-foldcase` 的纯 Scheme 实现
   - `string-ci=?`, `string-ci<?`, `string-ci>?`, `string-ci<=?`, `string-ci>=?` 改为基于 `string-foldcase` 的纯 Scheme 实现
   - 移除对 S7 C 内置实现的依赖（删除 `s7-char-ci=?`、`s7-string-ci=?` 等别名绑定）
2. 从 `src/s7_scheme_char.c` 和 `src/s7_scheme_char.h` 中移除 char-ci 的全部 C 实现：
   - `char_to_upper`、`g_char_cmp_ci`、`g_char_cmp_ci_not`
   - `g_chars_are_ci_*` 系列函数
   - `char_ci_*_b_*` 系列函数
3. 从 `src/s7.c` 中移除 char-ci 和 string-ci 的全部 C 层引用：
   - 符号字段定义（`char_ci_eq_symbol`、`string_ci_eq_symbol` 等）
   - hash-table 相关的 `char_ci_eq_hash_map`、`string_ci_eq_hash_map` 及 `hash_ci_char` 检查器
   - `s7_define_typed_function` 注册（10 个函数）
   - 优化器中的 `b_7pp`/`b_pp_unchecked` 绑定
   - `set_function_chooser` 引用
   - `main_names` 列表中的引用

### Why
- `char-foldcase` 和 `string-foldcase` 已在 Scheme 层实现了完整的 Unicode simple casing 支持
- char-ci 和 string-ci 函数应当与 foldcase 行为一致，都支持 Unicode
- 移除 C 实现可以减少核心解释器的复杂度，统一由 Scheme 层维护

### How
1. **char-ci 实现**：先对两个参数调用 `char-foldcase`，再用对应的 `char=?`/`char<?` 等函数比较；多参数场景用循环逐对比较 foldcase 后的结果
2. **string-ci 实现**：先对两个参数调用 `string-foldcase`，再用对应的 `string=?`/`string<?` 等函数比较；多参数场景同样用循环处理
3. **类型检查保留**：保持原有的 `char?`/`string?` 参数检查，错误信息不变
4. **C 层清理**：参考 `[0030]`（移除 string-upcase/string-downcase/string-foldcase 的 C 实现）的模式，从 s7.c 中彻底移除符号注册和优化器引用
