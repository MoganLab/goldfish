## 2026/05/03 (liii han-zi) 数值语义 API 定稿

### What
定稿并实现 `(liii han-zi)` 模块的数值语义层，提供单字符级别的汉字与数值互转 API。

1. 实现 `han-zi->number`：将单个汉字字符（通用/财务/年份风格）映射为非负整数
2. 实现 `number->han-zi`：将非负整数按指定风格（`'common` / `'financial` / `'year`）映射为单个汉字字符
3. 实现 `han-zi-number?`：判断单个字符是否属于本模块管辖范围内的数字汉字
4. 数值语义边界止于 **亿（10⁸）**，更大单位返回 `#f`
5. `'year` 风格仅支持 0–9，用于年份逐位表示（如二〇二六）

### Why
- 为后续 `(liii han-ci)` 等字符串级中文数字解析模块提供单字符原子能力
- 将字符级映射与字符串级组合逻辑解耦，保持 `(liii han-zi)` 职责单一

### How

#### han-zi->number 语义

| 字符范围 | 返回值 | 说明 |
|---------|--------|------|
| `〇` | 0 | 年份风格 |
| `零` `壹`–`玖` `拾` `佰` `仟` | 对应数值 | 财务风格 |
| `一`–`九` `十` `百` `千` `万` `亿` | 对应数值 | 通用风格 |
| `兆` 及更大单位 / 非数字汉字 | `#f` | 超出范围或不识别 |

三种风格一视同仁：`#\一`、`#\壹` 都返回 1。

#### number->han-zi 语义

| 数值 | `'common` | `'financial` | `'year` |
|------|-----------|--------------|---------|
| 0 | `零` | `零` | `〇` |
| 1–9 | `一`…`九` | `壹`…`玖` | `一`…`九` |
| 10 | `十` | `拾` | `#f` |
| 100 | `百` | `佰` | `#f` |
| 1000 | `千` | `仟` | `#f` |
| 10000 | `万` | `万` | `#f` |
| 100000000 | `亿` | `亿` | `#f` |
| > 10⁸ 或 < 0 | `#f` | `#f` | `#f` |

#### han-zi-number? 语义

| 输入 | 返回值 | 说明 |
|------|--------|------|
| `一` `壹` `〇` `亿` 等 | `#t` | `han-zi->number` 能识别的字符 |
| `兆` | `#f` | 超出亿范围，不算数字汉字 |
| `中` `A` `1` | `#f` | 非数字汉字 |
| 非字符类型 | `#f` 或类型错误 | Scheme 惯用做法 |

`han-zi-number?` 的边界与 `han-zi->number` 完全一致：到 `#\亿`（亿）为止，`#\兆`（兆）及以上返回 `#f`。

#### 边界声明

- **单位上限**：`#\亿`（10⁸）。`#\兆` 及更大单位返回 `#f`。
- **`'year` 风格**：仅支持 0–9，10 及以上返回 `#f`。
- **负值/小数/分数**：一律返回 `#f`，本模块只处理非负整数单字符映射。

### 已知问题
`bin/gf fmt` 会破坏 Unicode 字符字面量（如 `#\六`、`#\八`）。因此 `goldfish/liii/han-zi.scm` 和 `tests/liii/han-zi/` 下的测试文件不应运行 `bin/gf fmt`。

#### 与后续模块的分界

字符串级的 "一千一百" → 1100 属于 `(liii han-ci)` 或后续扩展模块，不在本任务范围内。
