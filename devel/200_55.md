# [200_55] 新增 gf source 子命令，用于直接查看库源码

## 如何测试

### 1. 查看现有库的源码
```
./bin/gf source liii/string
```
预期会直接输出 `(liii string)` 对应的源代码，开头类似
```
;
; Copyright (C) 2024 The Goldfish Scheme Authors
;
...
(define-library (liii string)
```

### 2. 查看 scheme 标准库源码
```
./bin/gf source scheme/base
```
预期会直接输出 `(scheme base)` 对应的源代码，开头类似
```
(define-library (scheme base)
```

### 3. 查询不存在的库
```
./bin/gf source liii/not-a-real-library
```
预期会输出
```
Error: library not found in *load-path*: liii/not-a-real-library
```

### 4. 查看帮助信息
```
./bin/gf help
```
预期在 Commands 区域看到
```
source ORG/LIB     Display exact library source from *load-path*
```

### 5. 相关测试
```
./bin/gf test tests/liii/goldsource
```
