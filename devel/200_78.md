## 2026/04/28 修复 quote / quasiquote 内部 `*newline*` 标记与 head-aware reader 排版

### What
修复 `gf fmt` 在格式化文件时，quote / quasiquote 内部的 `(*newline* n)` marker 被当成普通 datum 输出的问题，并让超出 `max-inline-length` 的 reader datum 进入接近普通 formatter 风格的多行排版。

本次修改只处理 `*newline*`，不处理 `*comment*`。

主要完成了以下工作：

1. 新增 reader datum 层面的 `(*newline* n)` 识别与展开。
2. 为 quote / quasiquote 内部 reader datum 增加带缩进的格式化入口。
3. 先生成 inline candidate，只有无 newline marker 且长度不超过 `max-inline-length` 时保持单行。
4. 多行 layout 按 `node-rules.json` 的 head 规则选择第一行参数和后续缩进。
5. 修复 `lambda` / `if` / `unless` / `map` 等 reader list 被逐元素拆坏的问题。
6. 保留原有单参数 `format-reader-datum` 入口，不使用 `define*`，避免 `:trie` 等 keyword datum 被误当成关键字参数。
7. 增加基于 `scan-file` 资源文件的回归测试，覆盖真实 tokenizer 注入 marker 的路径。

### Why
`scan-file` 会把源文件中的空行转换成内部 marker：

```scheme
(*newline* n)
```

普通顶层或普通 env 中的 newline marker 会被 formatter 识别并输出为空行。但 quote / quasiquote 内部的数据不是普通 formatter env 子树，而是 reader datum。旧实现的 reader datum formatter 只按普通 list 输出，所以会把 marker 原样打印出来：

```scheme
`(begin (*newline* 1) (define ,name ,value))
```

这会导致 `goldfish/scheme/base.scm` 等文件在格式化后出现不应该暴露的内部实现细节 `(*newline* n)`。

另外，长 quasiquote 如果只做“所有元素逐行拆开”，会得到不自然的结果：

```scheme
`(lambda
   ,new-args
   ,@(map
       (lambda
         (arg)
         (if
           ...
```

因此 reader datum 的多行 layout 需要理解 list head，至少要按普通 formatter 的 `firstLineLimit`、`allowFirstLineChildEnv` 和 `restIndent` 规则决定哪些元素留在第一行。

### How
修复集中在 `tools/fmt/liii/goldfmt-format.scm` 的 reader datum 输出层，不修改 tokenizer / scanner 架构。

#### 1. 严格识别 newline marker

`newline-marker-datum?` 只匹配形如 `(*newline* <number>)` 且刚好两个元素的 datum：

```scheme
(define (newline-marker-datum? datum)
  (and (pair? datum)
       (eq? (car datum) '*newline*)
       (pair? (cdr datum))
       (number? (cadr datum))
       (null? (cddr datum))))
```

这样不会误处理 `*comment*`，也不会误处理形态不完整或多参数的普通数据。

#### 2. 将 marker 展开成真实换行

`reader-newlines` 把 marker 中的空行数量转换为真实换行文本：

```scheme
(define (reader-newlines count)
  (let loop ((i (+ count 1))
             (result ""))
    (if (<= i 0)
        result
        (loop (- i 1) (string-append result "\n")))))
```

例如：

- `(*newline* 1)` 输出 `"\n\n"`，表示保留一个空行。
- `(*newline* 2)` 输出 `"\n\n\n"`，表示保留两个空行。

#### 3. reader datum 按条件选择 inline 或 multiline

`format-reader-pair-at` 先生成单行候选文本。只有同时满足下面条件才保持单行：

1. 内部没有 newline marker。
2. inline candidate 本身没有换行。
3. inline candidate 长度不超过 `max-inline-length`。

否则进入多行布局。

#### 4. 多行布局使用 head-aware 规则

`reader-head-name` 将 reader list 的 head 转成规则名；`reader-select-first-line-items` 和 `reader-rest-indent` 根据规则选择第一行参数和后续缩进。

这样可以让：

```scheme
`(lambda ,new-args
   ,@(map (lambda (arg)
            (if (pair? arg)
              ...
              (values)))
       args)
   ,@body)
```

而不是拆成：

```scheme
`(lambda
   ,new-args
   ,@(map
       (lambda
         (arg)
         (if
           ...
```

#### 5. quote / quasiquote 入口传入当前列

quote 输出 `'` 后，内部 datum 的缩进基准从 quote 后面开始：

```scheme
(emit-string! writer "'")
(emit-string! writer (format-inline-atom-or-quote-at content (+ column 1)))
```

quasiquote / unquote / unquote-splicing 由 reader datum formatter 内部递归处理：

```scheme
((single-arg-symbol-form? datum 'quasiquote)
 (string-append "`" (format-reader-datum-at (cadr datum) (+ indent 1))))
((single-arg-symbol-form? datum 'unquote)
 (string-append "," (format-reader-datum-at (cadr datum) (+ indent 1))))
((single-arg-symbol-form? datum 'unquote-splicing)
 (string-append ",@" (format-reader-datum-at (cadr datum) (+ indent 2))))
```

这样 `` ` ``、`,`、`,@` 的前缀宽度会参与缩进计算。

### 结果示例

输入：

```scheme
'(begin (*newline* 1) (define x 1))
```

输出：

```scheme
'(begin

   (define x 1))
```

长 quasiquote 超过 `max-inline-length` 后会展开为多行，同时保留常见 head 的第一行结构：

```scheme
`(lambda ,new-args
   ,@(map (lambda (arg)
            (if (pair? arg)
              `(unless (,(cadr arg) ,(car arg))
                 ...)
              (values)))
       args)
   ,@body)
```

普通短 reader datum 仍保持单行：

```scheme
`(a ,b)
'(a b c)
(:trie)
```

- 对 car 不是 symbol / syntax object 的 list，不再套用 head 规则；这类结构按 data list 处理，后续元素对齐首元素列，避免 quoted alist 首行混入两个元素。

- reader vector 也使用同一套 inline / multiline 判断；vector 内部的 newline marker 会展开成真实空行。
