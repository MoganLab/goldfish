## 2026/3/30 njson 测试按单函数拆分

### What
- 将 `tests/goldfish/liii/njson-test.scm` 拆为 `tests/liii/njson/<function>-test.scm` 的单函数测试集合。
- 新增入口说明文件 `tests/liii/njson-test.scm`，仅说明测试已迁移到目录内。
- 删除旧的聚合测试文件 `tests/goldfish/liii/njson-test.scm`。
- 统一采用当前仓库的命名约定：
  - `? -> -p`
  - `! -> -bang`
  - `-> -> -to-`
  - `= -> -eq`
  - `* -> -star`
  - `/ -> -slash-`
  - `< / > / <= / >= -> lt / gt / le / ge`
  - 单独运算符使用语义化文件名，如 `plus`、`minus`、`star`、`slash`
- 新文件统一补充中文注释块，包含函数名、简要说明、语法、参数、返回值、注意、示例和错误处理。
- 通过 `bin/gf fix` 统一生成右侧标签与格式。

### Why
- 原始 `njson-test.scm` 同时覆盖大量 API，容易把不同函数的行为混在一起，后续维护和回归定位都不够清晰。
- 单函数单文件后，每个测试文件只关注一个 API，能更明确地锁定语义和错误模型。
- 统一文档块和命名规则后，`njson` 测试风格可以和仓库里已有的单文件拆分保持一致。

### How
- 先按原聚合文件的章节顺序提取测试语义，再把每个 API 放进独立文件。
- 只保留对应函数自身的断言，辅助函数仅用于构造输入或捕获错误信息。
- 对需要异常断言的文件补齐 `(liii error)`、`(liii path)`、`(liii hash-table)`、`(rename (liii json) ...)` 等必要导入。
- 对需要格式化右侧标签的文件统一执行 `bin/gf fix tests/liii/njson`。
- 迁移后分别做两类验证：
  - 单文件验证：`bin/gf -m r7rs tests/liii/njson/<file>.scm`
  - 真实 runner 验证：`bin/gf test --only njson`
