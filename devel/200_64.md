# 直接在命令行流式处理下载内容
./bin/gf eval '(begin
  (import (liii http)
          (liii string))
  (define chunks (quote ()))
  (define r
    (http-get "https://httpbin.org/get"
      :stream #t
      :params (quote (("download" . "stream")))
      :output-file "/tmp/goldfish-http-download.txt"
      :callback (lambda (chunk userdata)
                  (when (> (string-length chunk) 0)
                    (set! chunks (cons chunk chunks)))
                  #t)))
  (write (r (quote status-code)))
  (newline)
  (write (> (length chunks) 0))
  (newline)
  (write (string-contains (string-join (reverse chunks) "") "stream"))
  (newline))'

      :params (quote (("chunk_size" . "8192") ("seed" . "3")))
      :callback (lambda (chunk userdata)
                  (set! total (+ total (string-length chunk)))
                  #t)))
  (write (r (quote status-code)))
  (newline)
  (write total)
  (newline))'

## 2026/04/07

### What

1. 在 `http-get` 中集成文件下载能力
   - 通过 `:output-file` 将响应体直接写入本地文件
   - 接口形式为：
     `http-get url [params] [headers] [proxy] [output-file] [stream] [callback] [userdata]`

2. 在 `http-get` 中集成流式下载能力
   - 通过 `:stream #t` 启用流式响应处理
   - 通过 `:callback` 按 chunk 处理响应体
   - 通过 `:userdata` 向回调透传用户数据

3. 保持 `http-get` 原有同步 GET 语义不变
   - 不传新参数时仍走普通 GET
   - 返回值仍然是统一的 response-object

4. 新增对应测试与示例
   - `tests/liii/http/http-get-test.scm`
   - `tests/liii/http-test.scm`

### Why

原来的 `http-get` 只适合把整个响应体一次性读入内存。

为了让接口更接近常见 HTTP 客户端库的使用方式，`http-get` 需要同时支持“直接下载到文件”和“边接收边处理”的工作流。这样就可以对标 `requests.get(..., stream=True)` 这类常见用法，也能避免大文件下载时必须把完整响应体保留在内存中。

### How

1. 在 `goldfish/liii/http.scm` 中扩展 `http-get`
   - 新增 `:output-file`、`:stream`、`:callback`、`:userdata`
   - 对新参数做类型校验和组合约束校验
   - `:stream #t` 时要求至少提供 `:callback` 或 `:output-file`

2. 在 `src/goldfish.hpp` 中直接扩展 `g_http-get`
   - 普通请求继续走 `session.Get()`
   - 当设置 `output-file` 或 `callback` 时改走 `session.Download(...)`
   - 下载过程中可同时写文件和执行 chunk 回调

3. 继续复用现有 HTTP 响应模型
   - 流式下载完成后仍返回统一的 response-object
   - `status-code`、`url`、`headers` 等访问方式保持不变
   - 下载路径下响应体已被流式消费，因此 `text` 为空字符串

4. 通过在线端点验证功能
   - 使用 `httpbin.org/get` 验证流式 chunk 回调
   - 使用 `jsonplaceholder.typicode.com/posts/1` 验证落盘下载
