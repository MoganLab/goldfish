## 2026/04/07

### What

1. 在 `http-get` 中集成文件下载能力
   - 通过 `:stream #t` + `:output-file` 将响应体按 chunk 流式写入本地文件
   - 接口形式为：
     `http-get url [params] [headers] [proxy] [output-file] [stream] [callback]`

2. 在 `http-get` 中集成按 chunk 的响应体处理能力
   - 通过 `:stream #t` + `:callback` 按 chunk 处理响应体

3. 保持 `http-get` 原有同步 GET 语义不变
   - 不传新参数时仍走普通 GET
   - 返回值仍然是统一的 response-object

4. 新增对应测试与示例
   - `tests/liii/http/http-get-test.scm`
   - `tests/liii/http-test.scm`
   - `demo/demo_http-get_file.scm`

### Why

原来的 `http-get` 只适合把整个响应体一次性读入内存。

为了让接口更接近常见 HTTP 客户端库的下载用法，同时尽量减少 C++ 侧专门为下载维护的代码，`http-get` 需要支持一条“在 Scheme 层通过 http-stream-get 做简单流式下载”的工作流。这样可以避免大文件下载时把完整响应体保留在内存中，并把简单下载逻辑上移到 `http.scm`。

### How

1. 在 `goldfish/liii/http.scm` 中扩展 `http-get`
   - 新增 `:output-file`、`:stream`、`:callback`
   - 对新参数做类型校验
   - 在 `:stream #t` 时通过 `http-stream-get` 处理简单流式下载
   - `:stream #t` 时要求至少提供 `:output-file` 或 `:callback`
   - `:stream #t` 时不支持 `headers`

2. 在 `src/goldfish.hpp` 中收缩下载相关实现
   - `g_http-get` 退回普通 GET 实现
   - `g_http-stream-get` / `g_http-stream-post` 继续负责流式 chunk 回调

3. 继续复用现有 HTTP 响应模型
   - 普通 `http-get` 仍返回统一的 response-object
   - `status-code`、`url`、`headers` 等访问方式保持不变
   - 简单流式下载模式则返回 `#<undefined>`

4. 通过在线端点验证功能
   - 使用 `httpbin.org/get` 验证流式 chunk 回调
   - 使用 `jsonplaceholder.typicode.com/posts/1` 验证 Scheme 层落盘下载
   - 使用 `proof.ovh.net/files/100Mb.dat` 验证大文件流式下载
