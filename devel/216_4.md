## 2026-03-30 拆分 alist-test.scm

### What
将 `tests/goldfish/liii/alist-test.scm` 按“一个函数一个测试文件”的方式迁移到 `tests/liii/alist/` 目录，并新增轻量入口说明文件 `tests/liii/alist-test.scm`。

本次拆分覆盖的函数如下：

1. `alist?` -> `alist-p-test.scm`
2. `alist-ref` -> `alist-ref-test.scm`
3. `alist-ref/default` -> `alist-ref-slash-default-test.scm`
4. `vector->alist` -> `vector-to-alist-test.scm`
5. `alist-cons` -> `alist-cons-test.scm`

### Why
1. 旧的 `alist-test.scm` 聚合了多个函数的测试，定位和维护成本较高。
2. 拆成单文件后，可以像 `list`、`vector` 目录一样单独运行和回归。
3. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
4. 统一每个测试文件的文档格式，便于后续继续迁移同类模块。

### How
1. 参考已有 `tests/liii/list/` 和 `tests/liii/vector/` 的拆分方式，按函数边界切分测试。
2. 对测试文件使用统一的注释结构：
   - 函数名
   - 简要说明
   - 语法
   - 参数
   - 返回值
   - 注意
   - 示例
   - 错误处理
3. 文件命名按统一规则转换特殊符号：
   - `?` -> `-p`
   - `!` -> `-bang`
   - `->` -> `-to-`
   - `=` -> `-eq`
   - `*` -> `-star`
   - `alist-ref/default` 统一写作 `alist-ref-slash-default-test.scm`
4. 保留原有测试覆盖范围，尽量只调整目录结构和说明文字，不改测试语义。
5. 删除旧的 `tests/goldfish/liii/alist-test.scm` 聚合文件，并把 `alist-cons` 测试迁到 `tests/liii/alist/` 目录。
6. 迁移完成后，逐个运行新测试文件，并通过入口说明文件做一次正式加载验证，避免只在辅助脚本下通过。
