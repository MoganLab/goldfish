## 2026/05/04 任务规划

### What
1. 新增 `bin/gf code` 子命令，将 Claude Code 启动流程集成到 `gf` 工具链中：
   - 同步 `hooks/pre-commit` 到 `.git/hooks/pre-commit`
   - 可选检查当前分支是否为 `main`
   - 在 `main` 分支时自动 `git pull origin main`
   - 启动 Claude Code (`claude` 命令)
2. 在 `(liii path)` 中实现 `path-copy`（文件复制）和 `path-copy-into`（复制到目录），参考 Python 的 `Path.copy` 和 `Path.copy_into`。
3. 删除原 `(liii path)` 中复制 path 对象的 `path-copy`，改用 s7 内置的 `copy` 函数，`path-copy` 改为文件复制语义。
4. 在 C 层（`src/goldfish.hpp`）新增 `g_path-copy` 函数，使用 tbox 文件 API 实现二进制安全的文件复制。

### Why
- 将 Claude Code 启动流程集成为 `gf code` 子命令可以统一 CLI 入口，减少维护成本。
- `(liii path)` 缺少文件复制功能，`path-copy` 和 `path-copy-into` 是常见的路径操作需求，对标 Python `pathlib` 的 API 设计。
- 原 `path-copy` 复制的是 path 记录对象而非文件，与大多数用户直觉不符。s7 内置的 `copy` 函数已能正确处理 record 对象，因此直接删除旧的 `path-copy`，让 `path-copy` 自然地承担文件复制的语义。

### How
1. **`bin/gf code` 子命令**：
   - 在 `gfproject.json` 中注册 `code` 子命令，指向 `liii/goldcode` 模块。
   - 在 `tools/code/liii/goldcode.scm` 中实现 `run-goldcode`，使用 `(liii os)` 的 `os-call` 执行 git 命令，使用 `(liii path)` 的 `path-read-text`/`path-write-text` 或新的 `path-copy` 同步 hook 文件。
   - `launch-claude` 使用 `g_system "claude"` 启动 Claude Code。

2. **`bin/gf code` 子命令**：
   - 不检查当前分支，直接执行以下流程：
     1. 同步 `hooks/pre-commit` 到 `.git/hooks/pre-commit`
     2. 若当前在 `main` 分支则 `git pull origin main`，否则跳过 pull
     3. 启动 Claude Code

3. **`path-copy` 与 `path-copy-into`**：
   - C 层：在 `src/goldfish.hpp` 的 `glue_liii_path` 中新增 `glue_path_copy`，函数 `f_path_copy` 使用 `tb_file_init`、`tb_file_read`、`tb_file_writ` 实现源文件到目标文件的逐块复制。
   - Scheme 层：在 `goldfish/liii/path.scm` 中导出 `path-copy`、`path-copy-into`。
     - `path-copy`：调用 `g_path-copy`，复制文件内容。
     - `path-copy-into`：提取源文件文件名，拼接目标目录，再调用 `path-copy`。
     - `path` 构造函数中传入 path 对象时，直接使用 s7 内置的 `copy` 函数复制 record。

3. **测试迁移**：
   - 删除 `tests/liii/path/path-copy-test.scm` 中原有测试（旧 `path-copy` 已移除）。
   - 新增 `path-copy-test.scm` 和 `path-copy-into-test.scm`，覆盖文件复制的基本场景和目录复制场景。
