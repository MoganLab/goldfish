## 2026/04/28 为 fmt / test 增加 changed-since 模式

### What
为 `gf fmt` 和 `gf test` 新增类似 Pants `--changed-since` 的参数：

```bash
bin/gf fmt --changed-since=HEAD
bin/gf fmt --dry-run --changed-since=HEAD
bin/gf test --changed-since=HEAD
```

本次只实现 `--changed-since`，不实现 Pants 的 `--changed-dependents`。

行为边界如下：

- `gf fmt --changed-since=REV` 只格式化从 `REV` 到当前工作区发生变化的、仍然存在的 `.scm` 文件。
- `gf fmt --changed-since=REV path` 会把 `path` 作为 git diff 的 pathspec，只处理该路径范围内的变更 `.scm` 文件。
- `gf fmt --dry-run --changed-since=REV` 只预览变更文件的格式化结果，不写回文件。
- `gf test --changed-since=REV` 只运行直接发生变化的 `*-test.scm` 测试文件。
- `gf test path-or-pattern --changed-since=REV` 会先按原有 path / pattern 规则过滤测试文件，再和 changed-since 结果取交集。
- 未跟踪文件不包含在结果里，因为底层使用的是 `git diff --name-only`。
- 源文件变更不会反推出依赖测试文件；这正是“不实现 `--changed-dependents`”的含义。

### Why
修改前，`gf fmt` 和 `gf test` 都只能面向显式路径、目录或全部测试运行：

```bash
bin/gf fmt goldfish
bin/gf test
bin/gf test tools/test/tests
```

在日常开发中，经常只需要处理相对于某个 git revision 的变更文件。例如：

```bash
bin/gf fmt --dry-run --changed-since=HEAD
bin/gf test --changed-since=HEAD
```

这样可以减少无关文件扫描和测试运行，特别适合提交前检查、CI 增量检查、以及开发过程中快速验证当前改动。

### How
#### 1. 新增 tools/common 公共模块

`tools/common/liii/goldtool-changed.scm` 提供三个函数：

```scheme
(changed-files-since since)
(changed-existing-files-since since)
(changed-scheme-files-since since)
```

也支持传入可选 path scope：

```scheme
(changed-scheme-files-since "HEAD" "tools/fmt")
```

核心实现是调用 git：

```scheme
git diff --name-only --relative REV -- [path]
```

然后把输出按行拆分，过滤空行。`changed-scheme-files-since` 进一步保留 `.scm` 且当前仍存在的文件。

#### 2. 在 fmt / test 中加载 common 模块

当前 C++ tool loader 会把 `tools/<command>` 加入 `*load-path*`，但不会自动加入 `tools/common`。因此 `goldfmt.scm` 和 `goldtest.scm` 在 `define-library` 前补充了 common 目录到 `*load-path*`，之后就可以导入：

```scheme
(import (liii goldtool-changed))
```

这里同时兼容两类运行方式：

- 在项目根目录通过 `bin/gf fmt` / `bin/gf test` 运行。
- 在某个 tool 目录下运行该 tool 自己的测试。

#### 3. `gf fmt` 接入 changed-since

`make-fmt-arg-parser` 新增：

```scheme
(parser :add-argument
  '((name . "changed-since") (type . string)))
```

主入口读取该参数：

```scheme
(changed-since (parser 'changed-since))
```

如果存在 `changed-since`，就走 `format-changed-since`：

```scheme
(format-changed-since changed-since path-str dry-run)
```

`format-changed-since` 的职责是：

1. 校验可选 path scope 是否存在。
2. 调用 `changed-scheme-files-since` 得到变更 `.scm` 文件。
3. 没有匹配文件时输出提示并成功返回。
4. 有匹配文件时复用 `format-file-list` 逐个格式化。
5. `--dry-run` 时只输出格式化结果，不写回文件。

#### 4. `gf test` 接入 changed-since

`make-test-arg-parser` 同样新增 `changed-since` 参数。

为了不让 `--changed-since` 被误认为测试目标，`check-and-switch-to-target` 会跳过：

```scheme
--changed-since REV
--changed-since=REV
```

`run-goldtest` 中的流程变成：

1. 保留原有 `path / pattern / filename / dir` 分类逻辑。
2. 得到原本应该运行的测试文件集合。
3. 如果指定了 `--changed-since`，再调用 `filter-changed-test-files` 做交集过滤。
4. 最终只运行交集中的测试文件。

这保证了旧行为不变，同时支持组合用法：

```bash
bin/gf test tools/test/tests --changed-since=HEAD
bin/gf test parse --changed-since=HEAD
```

### Notes
- `--changed-since` 基于 git，因此需要在 git 工作区内运行。
- 底层使用 `git diff --name-only`，所以不会包含 untracked 文件。
- 本次不处理依赖反查，也不新增 `--changed-dependents`。
- 如果没有匹配的变更 Scheme 文件或测试文件，命令会输出提示并以成功状态结束。
