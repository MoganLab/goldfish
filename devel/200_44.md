## 2026-03-31 按照 R7RS 新增 (scheme eval) 库

### What
新增标准库文件：

1. `(scheme eval)`

对应提供的导出为：

1. `environment`
2. `eval`

### Why
Goldfish 底层已经具备 `eval` 和环境对象等相关能力，但此前缺少标准 R7RS 风格的 `(scheme eval)` 库入口。

这会带来两个问题：

1. R7RS 风格代码无法直接写出：
   ```scheme
   (import (scheme eval))
   ```
2. 依赖 `environment` + `eval` 组合能力的代码，在移植到 Goldfish 时需要改写为实现相关写法

因此这次改动的目标不是重写一套求值器，而是把现有能力整理成标准库层面的可导入接口。

### How
本次实现遵循“尽量只做 re-export，不要新写代码”的原则：

1. 新增 `goldfish/scheme/eval.scm`
2. 直接导出底层已有的 `eval`
3. `environment` 只补一个最薄的适配层

其中 `environment` 的实现方式是：

1. 先创建一个新的 `inlet`
2. 如果传入了 import set，就在这个环境中执行一次 `(import ...)`
3. 返回该环境对象，供后续 `eval` 复用

也就是说，这次没有自己重新解析 `only`、`except`、`prefix`、`rename` 等 import set 语义，而是直接复用现有库系统中的 `import` 行为。

同时，这次实现没有修改 `boot.scm`，只是在标准库层增加了一个新的 R7RS 入口。

### 测试补充
本次同时新增 `(scheme eval)` 的对应测试，并按照当前 `tests/scheme` 目录的风格拆分为：

1. `tests/scheme/eval-test.scm`
2. `tests/scheme/eval/environment-test.scm`
3. `tests/scheme/eval/eval-test.scm`

其中：

1. `tests/scheme/eval-test.scm` 作为目录入口说明文件
2. `environment-test.scm` 负责验证 `environment` 构造环境、导入多个库以及 `only` / `except` / `prefix` / `rename` 等 import set 行为
3. `eval-test.scm` 负责验证 `eval` 的基本求值、同一环境下的增量求值，以及未绑定变量错误

测试风格也与当前 `tests/scheme` 下其他标准库测试保持一致，不再使用旧的单文件 `tests/goldfish/...` 组织方式。
