## 2026/04/08

### What

1. 按当前 SRFI-267 规范重新整理 raw string 实现
   - 在 `goldfish/srfi/srfi-267.scm` 中补齐并导出：
     `raw-string-read-error?`
     `raw-string-write-error?`
     `read-raw-string`
     `read-raw-string-after-prefix`
     `can-delimit?`
     `generate-delimiter`
     `write-raw-string`
   - 保留 `#"` reader 扩展，使 `#"..."` raw string 字面量仍可直接读取
   - 修正 delimiter 读取逻辑，使其可以处理带空格、括号等字符的 delimiter

2. 基于新的 SRFI-267 实现整理 `(liii raw-string)`
   - `(liii raw-string)` 直接基于 `(srfi srfi-267)` 提供 raw string 能力
   - 在保留 `deindent` / `&-` 的同时，对外重导出 SRFI-267 的公共过程
   - 继续沿用 C# 风格的多行字符串去缩进规则

3. 将 `tests/liii/raw-string-test.scm` 改为顶层入口文件
   - 不再把它作为一个混合断言测试文件
   - 改写为“模块说明 + 示例 + 函数分类索引”的导览页
   - 对齐 `devel/Unit_Test_ZH.md` 中对顶层 `<library>-test.scm` 的约定

4. 将实际断言测试拆到 `tests/liii/raw-string/` 子目录
   - `raw-string-literal-test.scm`：测试 `#"...` raw string 字面量
   - `deindent-test.scm`：测试 `deindent`
   - `ampersand-minus-test.scm`：测试 `&-`
   - 每个文件只围绕一个测试目标组织说明块和断言

5. 新增 `tests/srfi/srfi-267-test.scm`
   - 覆盖 SRFI-267 过程接口、错误谓词、delimiter 相关行为和 reader 语法
   - 用于验证 SRFI 层实现与 `(liii raw-string)` 封装之间没有回退

### Why

1. 之前的 raw string 实现更偏向“reader 子集”，和当前 SRFI-267 页面上的公共过程接口并不完全一致。

2. `(liii raw-string)` 顶层测试文件同时混合了模块入口说明、reader 语法、`deindent` 和 `&-` 的大量断言，不符合当前 `tests/` 目录约定的“两层结构”。

3. `devel/Unit_Test_ZH.md` 已经明确：
   - 顶层 `tests/<group>/<library>-test.scm` 应该是入口说明文件
   - 具体断言测试应拆到 `tests/<group>/<library>/` 子目录
   - 单个测试文件应围绕一个目标展开

4. raw string 属于语法入口 + 过程 API + 宏工具同时存在的一组能力，如果不拆开测试，后续维护时很难判断回归到底发生在哪一层。

### How

1. 在 `goldfish/srfi/srfi-267.scm` 中重写核心实现
   - 把 reader hook 和公共过程接口拆开处理
   - `read-raw-string-after-prefix` / `read-raw-string` 面向用户暴露标准过程
   - `reader-read-raw-string` 负责和 `*#readers*` 集成
   - `read-raw-body` 负责真正的 delimiter 匹配与正文读取

2. 在 `(liii raw-string)` 中复用 SRFI 层能力
   - 将 SRFI-267 读写过程直接导出给 `(liii raw-string)` 用户
   - 把 `deindent` / `&-` 保留在 `(liii raw-string)` 这一层
   - 让“raw string 读写”和“多行缩进处理”职责边界更清晰

3. 按 `Unit_Test_ZH.md` 重组测试目录
   - 顶层 `tests/liii/raw-string-test.scm` 只保留导览职责
   - 新建 `tests/liii/raw-string/` 子目录承载断言测试
   - 在子测试文件中统一采用：
     `import`
     `check-set-mode!`
     说明块
     `check` / `check-catch`
     `check-report`

4. 保留并补强验证范围
   - raw string 字面量测试覆盖空串、反斜杠、双引号、多行、注释文本和自定义 delimiter
   - `deindent` / `&-` 测试覆盖正常缩进、空行、行尾空格、Unicode 和错误路径
   - SRFI 测试覆盖 delimiter 生成、写出、读取和错误谓词
