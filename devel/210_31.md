## 2026-03-27 新增 `(liii string)` 的 `string-replace`

### What
在 `(liii string)` 中新增顶层函数：

```scheme
(string-replace str old new)
```

语义定义为：

1. 默认替换 `str` 中所有 `old` 的非重叠匹配
2. 按原字符串从左到右扫描
3. 不会重复扫描刚插入的 `new`
4. `old` 为空字符串时，返回 `str` 的副本
5. 没有匹配时，返回原内容的副本

例如：

```scheme
(string-replace "hello world hello" "hello" "hi")
;; => "hi world hi"

(string-replace "aaa" "a" "aa")
;; => "aaaaaa"
```

### Why
`srfi-13` 本身也有一个 `string-replace`，但它的语义更接近“按区间拼接子串”，并不符合大多数开发者和 AI Coding 场景下对 `replace` 的直觉。

在日常编码里，看到：

```scheme
(string-replace str old new)
```

通常会自然理解为：

- 查找 `old`
- 把所有匹配都替换成 `new`

如果继续直接暴露 `srfi-13` 的原始语义，AI 或使用者很容易写出看起来正确、实际行为却完全不同的代码。

### How
实现方式如下：

1. 在 `(liii string)` 的 import 中显式排除 `srfi-13` 的 `string-replace`
2. 导出 Goldfish 自己的 `string-replace`
3. 使用 `typed-lambda` 保证三个参数都必须是字符串
4. 使用 `string-position` 在原字符串中逐段查找 `old`
5. 通过累积片段后一次性 `string-append`，构造最终结果
6. 对 `old = ""` 特判，直接返回 `string-copy str`，避免无穷替换

### 测试覆盖
本次新增测试覆盖以下场景：

1. 基本的全部替换
2. 未命中时返回原内容
3. `new = ""` 的删除语义
4. `old = ""` 的边界行为
5. 非重叠替换
6. `new` 中再次包含 `old` 时不重复扫描
7. Unicode 字符串替换
8. 类型错误与参数个数错误
