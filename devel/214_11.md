## 2026/3/2 njson 新增merge api


### What
1. njson merge 能力新增（C++ 层）：
   - 在 `src/goldfish.hpp` 新增 `njson_run_merge`，支持 shallow/deep 两种模式；
   - 新增并注册 `g_njson-merge` / `g_njson-merge!` / `g_njson-deep-merge` / `g_njson-deep-merge!`；
   - merge! / deep-merge! 成功后会触发 keys 缓存失效。
2. njson merge API 暴露（Scheme 层）：
   - `goldfish/liii/njson.scm` 导出并封装 `njson-merge` / `njson-merge!` / `njson-deep-merge` / `njson-deep-merge!`；
   - 新增 `njson%%object-handle?`；
   - 当前分支口径下，`json` 与 `other` 均要求为 `njson object-handle`。
3. njson 测试与文档扩展：
   - `tests/goldfish/liii/njson-test.scm` 新增 merge/deep-merge 章节与用例；
   - 覆盖函数式与原地语义、对象覆盖/递归合并、释放句柄、非对象/非句柄入参错误断言；
   - 当前断言与 Scheme 层收紧后的错误消息一致（`json/other must be njson object-handle`）。

### Why
1. njson 缺少标准 merge/deep-merge 能力，调用方需要在业务层重复拼装 object 合并逻辑。
2. merge 的入参若放宽到标量，会带来“先通过参数检查、再在底层报 object 限制”的双层语义，接口可预期性较差。
3. 需要通过独立测试块把 shallow/deep、函数式/原地式、错误路径都锁定成可回归行为。

### How
1. C++：新增 `njson_merge_mode` 枚举与 `njson_run_merge` 公共执行路径。
2. C++：在执行前校验 source 可转为 JSON，并要求 source/target 都为 object；根据模式调用 `json::update(..., merge_objects)`。
3. C++：新增 4 个 glue 函数并在 `glue_njson` 注册 Scheme 可见符号。
4. Scheme：新增 merge 系列导出与 wrapper。
5. Scheme：通过 `njson%%object-handle?` 前置收紧入参，保证 `json` 和 `other` 都必须是 njson 对象句柄。
6. Test：新增 merge/deep-merge 文档化说明和功能/异常用例，错误消息断言同步到最新口径。

### 兼容性说明
1. 新增 API：`njson-merge` / `njson-merge!` / `njson-deep-merge` / `njson-deep-merge!`。
2. 当前分支语义下，merge 系列不再接受 strict scalar 作为 `other`；`json` 与 `other` 都必须是 njson 对象句柄。
