## 2026-05-28 迁移 s7.c 中剩余三角函数

### What
将 s7.c 中尚未迁移的双曲三角函数及相关数学函数迁移到 s7_scheme_inexact.c，与已有三角函数集中管理。

具体迁移函数列表：
1. **双曲函数**：sinh, cosh, tanh, asinh, acosh, atanh
2. **指数运算**：expt

每个函数包含：
- `*_p_p` — 核心实现（s7_pointer -> s7_pointer）
- `g_*` — Scheme 入口（args 列表）
- `*_p_d` / `*_d_d` — 优化器辅助函数（如适用）
- 辅助函数（如 int_to_int, int_pow_ok 等）

### Why
- s7.c 体积过大，需要继续拆分
- sin/cos/tan/asin/acos/atan/sqrt/exp/log 已在 s7_scheme_inexact.c 中，将同类函数集中管理符合已有架构
- 减少 s7.c 中直接操作内部宏（type/integer/make_real 等）的代码，统一使用公开 API

### How

#### 1. 目标文件选择
**推荐迁移到 `src/s7_scheme_inexact.c`**，理由：
- sin/cos/tan/asin/acos/atan/sqrt/exp/log 已在此文件中
- magnitude/angle/complex 在 s7_scheme_complex.c 中（复数构造相关）
- 双曲函数与三角函数数学本质同类，应集中管理

若项目策略要求所有新增 (scheme base) 函数必须进入 s7_scheme_base.c，则改为迁移到 s7_scheme_base.c。

#### 2. s7.c 内部 API 替换对照表

| s7.c 内部用法 | 迁移后替换为 |
|---|---|
| `type(x) == T_INTEGER` | `s7_is_integer(x)` |
| `type(x) == T_RATIO` | `s7_is_rational(x) && !s7_is_integer(x)` |
| `type(x) == T_REAL` | `s7_is_real(x)` |
| `type(x) == T_COMPLEX` | `s7_is_complex(x)` |
| `integer(x)` | `s7_integer(x)` |
| `fraction(x)` | `(double)s7_numerator(x) / (double)s7_denominator(x)` |
| `real(x)` | `s7_real(x)` |
| `real_part(x)` | `s7_real_part(x)` |
| `imag_part(x)` | `s7_imag_part(x)` |
| `make_real(sc, val)` | `s7_make_real(sc, val)` |
| `make_integer(sc, val)` | `s7_make_integer(sc, val)` |
| `int_zero` | `s7_make_integer(sc, 0)` |
| `int_one` | `s7_make_integer(sc, 1)` |
| `real_one` | `s7_make_real(sc, 1.0)` |
| `real_zero` | `s7_make_real(sc, 0.0)` |
| `c_complex_to_s7(sc, z)` | `s7_make_complex(sc, creal(z), cimag(z))` |
| `to_c_complex(x)` | `s7_real_part(x) + s7_imag_part(x) * _Complex_I` |
| `method_or_bust_p(sc, x, sym, str)` | `s7i_method_or_bust_p(sc, x, "func-name", str)` |
| `out_of_range_error_nr(...)` | `s7_out_of_range_error(...)` |
| `division_by_zero_error_2_nr(...)` | `s7_division_by_zero_error(sc, "expt", n, pw)` 或等效处理 |
