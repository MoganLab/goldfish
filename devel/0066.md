## 5. 迁移方案

### 5.1 需要迁移的代码

goldfish.hpp 中 HTTP 相关代码分布在行 116–699，具体包括：

**辅助函数**（行 116–260）：
- `error2hashtable` — 构建错误响应哈希表
- `response2hashtable` — 将 cpr::Response 转为哈希表
- `to_cpr_parameters` — 转换查询参数
- `to_cpr_headers` — 转换请求头
- `to_cpr_proxies` — 转换代理配置
- `to_cpr_multipart_part` — 转换单个多部分表单项
- `append_cpr_multipart_file_parts` — 追加文件部分
- `append_cpr_multipart_form_parts` — 追加表单部分
- `to_cpr_post_multipart` — 构建完整多部分请求体

**同步 HTTP**（行 262–402）：
- `f_http_head` / `glue_http_head`
- `f_http_get` / `glue_http_get`
- `f_http_post` / `glue_http_post`
- `glue_http`

**异步 HTTP**（行 404–699）：
- `AsyncHttpRequest` 结构体
- `g_async_requests_mutex` / `g_async_requests` 全局变量
- `process_async_http_callbacks`
- `f_http_async_get` / `glue_http_async_get`
- `f_http_async_post` / `glue_http_async_post`
- `f_http_async_head` / `glue_http_async_head`
- `f_http_poll` / `glue_http_poll`
- `f_http_wait_all` / `glue_http_wait_all`
- `glue_http_async`

### 5.2 迁移方式

参照 `src/liii_njson.cpp` 的模式：

1. **新建 `src/liii_http.cpp`**，放在 `namespace goldfish` 中
2. 按需引入头文件（`s7.h`、`cpr/cpr.h`、`<mutex>`、`<thread>`、`<chrono>`、`<vector>`、`<memory>` 等），不 `#include "goldfish.hpp"`
3. 辅助函数保持 `static`（仅在该编译单元使用）
4. 提供 `void glue_http(s7_scheme* sc)` 和 `void glue_http_async(s7_scheme* sc)` 两个入口函数，移除 `inline` 修饰
5. 在 `goldfish.hpp` 中删除 HTTP 实现代码，仅保留前向声明：
   - `void glue_http(s7_scheme* sc);`
   - `void glue_http_async(s7_scheme* sc);`
6. 在 `xmake.lua` 中添加 `add_files("src/liii_http.cpp")`

### 5.3 注意事项

- `glue_define` 辅助函数已在 `goldfish` 命名空间中定义（`goldfish.hpp` 和 `liii_njson.cpp` 均使用），`liii_http.cpp` 中也可以使用，需要在 `liii_http.cpp` 中重复定义
- 当前 HTTP glue 函数直接使用 `s7_define` 模式而非 `glue_define`，迁移时保持原有风格不变
- `goldfish.hpp` 中 `glue_http` 和 `glue_http_async` 的调用位置（行 2067–2068）不需要修改
