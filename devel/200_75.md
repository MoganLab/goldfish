# char, string 支持 Unicode

> 本文档遵循 [x_y.md](x_y.md) 中定义的格式规范。

## 完善char的unicode支持（`913e9889`）
## char?支持中文字符（`a34f16f8`）

## 实现 (liii string-cursor)

### What
新增 `(liii string-cursor)` 库，提供基于 cursor（游标）的字符串操作，使字符串操作支持 Unicode 字符级别语义，同时**不修改 `(liii string)` 的现有语义**。

`(liii string)` 保持字节级语义不变（`string-length` 返回字节数，`string-ref` 按字节索引），新增的 `(liii string-cursor)` 提供字符级语义：

- `string-cursor-start/end/next/prev/forward/back` — cursor 导航
- `string-cursor=?/</>/<=/>=?` — cursor 比较
- `string-cursor-diff` — 计算两个 cursor 之间的字符数
- `string-cursor->index` / `string-index->cursor` — cursor 与字符索引互转
- `string-ref/cursor` — 按 cursor 取完整 Unicode 字符
- `substring/cursors` — 按 cursor 截取子串
- `string-index/string-skip` — 搜索返回 cursor
- `string-take/string-drop` — 按字符数截取
- `string-fold/string-fold-right` — 字符级 fold

### Why
当前 `(liii string)` 对 UTF-8 字符串的操作是按字节而非按字符：

```scheme
(string-length "中文")    ; => 6  （字节长度）
(string-ref "中文" 0)     ; => #\xE4 （UTF-8 首字节，不是 #\中）
```

这导致：
1. 用户无法直观地对 Unicode 字符串做"取第 N 个字符"的操作
2. `string-split`、`string-take` 等函数在中文内容上会按字节切割，破坏字符编码
3. 与 `(liii unicode)` 提供的 `utf8-string-length`、`utf8-substring` 形成割裂

新增 `(liii string-cursor)` 是为了：
- 保持 `(liii string)` 向后兼容
- 提供一套完整的、字符级语义的字符串操作 API
- 遵循 SRFI-130（Cursor-based string library）规范

### How

#### cursor 的表示
cursor 是**不透明对象**，只含轻量数据（`char-index` + `offsets`），使用 `define-record-type` 定义：

```scheme
(define-record-type <string-offsets>
  (make-string-offsets bv positions)
  string-offsets?
  (bv string-offsets-bv)
  (positions string-offsets-positions))

(define-record-type <string-cursor>
  (make-string-cursor-raw offsets char-index)
  string-cursor?
  (offsets string-cursor-offsets)
  (char-index string-cursor-char-index))
```

字段说明：
- `offsets`：`<string-offsets>` 对象，内部封装 `bv`（字符串的 UTF-8 bytevector）和 `positions`（预扫描生成的位置向量）
- `char-index`：当前 cursor 指向的字符索引（0 到 n，n 为字符数）

`char-index` = n 时，表示 **post-end cursor**（最后一个字符之后的位置），与 SRFI-130 规定的 "n + 1 个有效 cursor" 一致。

**预扫描生成 `positions`**：
```scheme
(define (make-string-positions bv)
  (let ((len (bytevector-length bv)))
    (let loop ((pos 0) (result '(0)))
      (if (>= pos len)
          (list->vector (reverse result))
          (let ((next (bytevector-advance-utf8 bv pos len)))
            (loop next (cons next result)))))))
```
例如 `"我是中国人"`（假设每个字符 3 字节），`positions` 为 `#(0 3 6 9 12 15)`，长度 = 字符数 + 1（末位是 `bv` 总长，即 end cursor 位置）。

**为什么 cursor 内部不存 `str`**：
- SRFI-130 的函数签名中 `str` 已经是显式参数，cursor 存 `str` 是冗余
- cursor 只存 `char-index` 和 `offsets`，轻量；`bv` 绑定在 `offsets` 内部，供内部操作复用
- 所有导航操作只改 `char-index`，通过 `offsets` 查字节位置，均为 O(1)

#### 核心操作实现

1. **创建 cursor**（`string-cursor-start` / `string-cursor-end`）：
   ```scheme
   (define (string-cursor-start str)
     (let* ((bv (string->utf8 str))
            (off (make-string-offsets bv (make-string-positions bv))))
       (make-string-cursor-raw off 0)))

   (define (string-cursor-end str)
     (let* ((bv (string->utf8 str))
            (off (make-string-offsets bv (make-string-positions bv))))
       (make-string-cursor-raw off (- (vector-length (string-offsets-positions off)) 1))))
   ```
   创建时调用一次 `string->utf8` 和一次 `make-string-positions`（预扫描整串），`bv` 和 `positions` 封装进 `offsets` 对象。`string-cursor-end` 的 `char-index` 设为 n（post-end cursor）。

2. **向右导航**（`string-cursor-next`）— O(1)：
   `char-index` 直接 +1，通过 `offsets` 查新的 `byte-pos`。`char-index` = n 时不能再向右（已到达 post-end cursor）。

3. **向左导航**（`string-cursor-prev`）— O(1)：
   `char-index` 直接 -1，通过 `offsets` 查新的 `byte-pos`。

4. **cursor 转字符索引**（`string-cursor->index`）— O(1)：
   直接返回 `char-index`。

5. **字符索引转 cursor**（`string-index->cursor`）— O(1)：
   直接构造新 cursor，`char-index` 设为 n，通过 `offsets` 即可定位。

6. **取值函数**（`string-ref/cursor`）：
   ```scheme
   (define (string-ref/cursor str cursor)
     (let* ((off (string-cursor-offsets cursor))
            (bv (string-offsets-bv off))
            (pos (string-offsets-positions off))
            (idx (string-cursor-char-index cursor))
            (start (vector-ref pos idx))
            (end (vector-ref pos (+ idx 1)))
            (char-bv (bytevector-copy bv start end)))
       (string-ref (utf8->string char-bv) 0)))
   ```
   利用 `offsets` 直接定位字节边界，无需再次解析 UTF-8。

7. **截取子串**（`substring/cursors`）：
   通过两个 cursor 的 `char-index` 查 `offsets` 得到字节区间，再用 `bytevector-copy` + `utf8->string` 转回。

#### 性能要点

- **一次性预扫描**：`string-cursor-start` 创建时扫描整串一次，生成 `offsets` 向量。后续所有 cursor 操作均复用该 `offsets` 和 `bv`
- **所有导航 O(1)**：`string-cursor-next` / `string-cursor-prev` / `string-cursor-forward` / `string-cursor-back` 直接改 `char-index`，通过 `offsets` 查字节位置
- **`string-cursor->index` / `string-index->cursor` 均为 O(1)**：前者直接返回 `char-index`，后者直接构造 cursor
- **核心依赖 `(liii unicode)`**：`bytevector-advance-utf8`（仅用于预扫描生成 `offsets`）、`utf8->string`、`string->utf8`

#### 与 (liii string) 的关系

| 库 | 语义 | 示例 `"中文"` |
|---|---|---|
| `(liii string)` | 字节级 | `string-length` => 6, `string-ref` 取字节 |
| `(liii string-cursor)` | 字符级 | `string-cursor-diff` => 2, `string-ref/cursor` 取字符 |

两者独立，用户按需选用。
