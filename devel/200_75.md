   - `char->integer`：对 Unicode 字符返回正确的 codepoint（如 `#\中` → 20013）

### Why
- 第一步仅让 `char?` 能识别 Unicode 字符，但 char 相关操作函数仍只支持 ASCII
- 例如 `(char->integer #\中)` 仍返回错误结果（或仅返回低 8 位）
- `(char-upcase #\ä)` 无法正确返回 `#\Ä`
- `(char-alphabetic? #\中)` 返回 `#f`，但 Chez Scheme 返回 `#t`（汉字被视为 alphabetic）
- `(char-alphabetic? #\ä)` 返回 `#f`，无法正确识别拉丁字母变体

### How

#### 实现策略

**核心原则：先查 Chez Scheme 行为，再写测试，最后实现。**

每步实施流程：
1. 在 Chez Scheme 中验证目标函数的 Unicode 行为
2. 根据 Chez Scheme 的行为编写/更新测试用例
3. 修改 `src/s7_scheme_char.c` 中的实现
4. 运行对应测试文件验证
5. 运行 `bin/gf test` 全量测试通过后再 commit

**使用 Chez Scheme 验证行为的方法：**

创建临时 Scheme 文件（如 `/tmp/chez-test.scm`）：
```scheme
(display (char->integer #\中))
(newline)
(display (char-alphabetic? #\中))
(newline)
(display (char-upcase #\ä))
(newline)
```

然后运行：
```bash
/usr/bin/chezscheme --script /tmp/chez-test.scm
```

输出示例：
```
20013
#t
Ä
```

对于需要大小写/分类判断的函数，采用以下策略：

1. **ASCII 字符（codepoint < 256）**：继续使用现有的静态数组（`uppers`、`lowers`、`char_is_alphabetic` 等），保证性能和向后兼容
2. **Unicode 字符（codepoint >= 256）**：
   - 使用 C 标准库的宽字符函数：`towupper`/`towlower`、`iswalpha`/`iswdigit`/`iswspace`/`iswupper`/`iswlower`
   - 需要将 `uint32_t` codepoint 转换为 `wchar_t` 后调用
   - 注意：宽字符函数的 Unicode 支持取决于系统 locale 设置，在大多数现代系统上（Linux/macOS/Windows）支持 BMP（基本多文种平面，U+0000 到 U+FFFF）
   - 对于超出 BMP 的字符（如 emoji），宽字符函数可能返回原字符（不做转换），这是可接受的行为

**Chez Scheme 实际验证结果：**

```bash
$ /usr/bin/chezscheme --script /tmp/chez-test.scm
20013        ; (char->integer #\中)
#t           ; (char-alphabetic? #\中)  ← 注意：Chez 对汉字返回 #t
#t           ; (char-alphabetic? #\ä)
Ä            ; (char-upcase #\ä)
#t           ; (char-ci=? #\ä #\Ä)
#t           ; (char-numeric? #\１) 全角数字
#t           ; (char-whitespace? #\　) 全角空格 U+3000
中            ; (integer->char 20013)
```

**重要发现：**
- `(char-alphabetic? #\中)` 在 Chez Scheme 中返回 `#t`，不是 `#f`
- `(char-numeric? #\１)`（全角数字 U+FF11）返回 `#t`
- `(char-whitespace? #\　)`（全角空格 U+3000）返回 `#t`
- 汉字在 Chez Scheme 中被视为 alphabetic，这与 C 标准库的 `iswalpha` 行为一致
