### 注释格式
每个拆分后的测试文件都采用统一的文档注释格式：函数名、简要说明、`语法 / 参数 / 返回值 / 注意 / 示例 / 错误处理`。

### 文件命名规则
本次迁移沿用统一的特殊符号转写规则：

1. `?` -> `-p`
2. `!` -> `-bang`
3. `->` -> `-to-`
4. `=` -> `-eq`
5. `*` -> `-star`
6. `/` -> `-slash-`
7. 关系运算符中的 `<` / `>` / `<=` / `>=` 按语义写法转写

### Why
1. 原 `tests/goldfish/liii/enum-test.scm` 聚合了大量独立 API 的测试，单文件维护成本偏高。
2. 迁移到 `tests/liii/enum/` 后，可以按函数维度单独执行和回归。
3. 与 `string` / `vector` / `ascii` / `base` / `either` 的迁移方式保持一致。
4. 每个测试文件只断言对应函数的行为，避免不同 API 的断言互相混杂。

### How
1. 从原 `enum-test.scm` 中按函数边界提取测试代码。
2. 将每个函数拆成独立测试文件，并保留原有测试覆盖范围。
3. 为每个新文件补齐统一的文档块注释。
4. 新增 `tests/liii/enum-test.scm` 作为迁移后的入口说明文件。
5. 删除旧的聚合测试文件 `tests/goldfish/liii/enum-test.scm`。
6. 使用项目真实测试入口 `bin/goldfish -m r7rs` 逐个执行 `tests/liii/enum/` 下的所有测试文件，确认迁移后的测试可以独立通过。
