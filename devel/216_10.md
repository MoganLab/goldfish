### Why
1. 旧的 `enum-test.scm` 聚合了大量谓词、查找、集合操作和兼容性过程，不利于维护和回归。
2. 拆成单文件后，可以针对某个导出项单独运行，排查更快。
3. `enum` 还包含 `!` 版本和 R6RS 兼容接口，单文件化后更容易保持覆盖边界清晰。
4. 入口说明文件保留模块级导航，同时避免旧聚合文件继续承担测试职责。
5. 每个测试文件只验证自己对应的导出项，其他函数只允许作为构造输入或辅助断言，不再承载别的函数的正确性检查。

### How
1. 参考已有 `tests/liii/list/`、`tests/liii/vector/`、`tests/liii/alist/` 的迁移方式，按导出项边界切分测试。
2. 每个新文件都保留统一的注释结构：
   - 函数名
   - 简要说明
   - 语法
   - 参数
   - 返回值
   - 注意
   - 示例
   - 错误处理
3. 文件命名按统一规则转换特殊符号：
   - `?` -> `-p`
   - `!` -> `-bang`
   - `->` -> `-to-`
   - `=` -> `-eq`
   - `*` -> `-star`
   - `/` -> `-slash-`
   - `enum<?` / `enum>?` / `enum<=?` / `enum>=?` 以及对应的 `enum-set` 版本，沿用项目里现有的 `lt/gt/le/ge` 文件命名习惯
4. 为保证每个测试文件可独立运行，统一前置原始测试里的共享 fixture：
   - `color-names` / `color` / `color-set`
   - `reddish` / `~reddish`
   - `pizza-descriptions` / `pizza`
   - `always` / `never`
   - `fresh-sets`
5. 保留原有测试覆盖范围，尽量只做结构迁移，不随意改测试语义。
6. 删除旧的 `tests/goldfish/liii/enum-test.scm` 聚合文件。
7. 修正了拆分过程中一个错位文件：`enum-set-test.scm` 现在只测试 `enum-set`，不再混入 `enum-set-universe` 的断言。
8. 迁移完成后，逐个运行新测试文件，并通过入口说明文件做一次正式加载验证，避免只在辅助脚本下通过。
