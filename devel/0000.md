## 2026/05/02 重新实现 char-upcase 和 char-downcase
### What
按照 scheme.exe (Chez Scheme) 的 Unicode simple casing 行为，在 `(scheme char)` 的 Scheme 层重新实现 `char-upcase` 和 `char-downcase`。

1. 使用 Chez Scheme 10.3.0 生成 Unicode simple casing 映射数据
2. 在 `goldfish/scheme/char.scm` 中实现基于 Unicode 映射表的 `char-upcase` 和 `char-downcase`
3. 移除 `s7-char-upcase` 和 `s7-char-downcase` 的引用
4. 删除 C 语言实现 `g_char_upcase`、`g_char_downcase` 及相关代码

### Why
- 当前 C 实现依赖 `towupper`/`towlower`，在不同平台（特别是 Windows 和 macOS）行为可能不一致
- Scheme 层实现可以精确控制 Unicode simple casing 行为，与 Chez Scheme 保持一致

### How
1. **数据提取**：使用 Chez Scheme 10.3.0 遍历所有有效 Unicode 码点（跳过代理区 `#xD800`-`#xDFFF`），收集 `char-upcase` 和 `char-downcase` 的映射结果。共得到约 1400 个有 casing 变化的字符。
2. **范围合并**：将连续的、偏移量相同的码点合并为范围。合并后 upcase 约 690 个范围，downcase 约 674 个范围。
3. **Scheme 实现**：使用 `cond` 表达式（而非 `case`），因为 `cond` 支持范围检查 `(<= #x61 cp #x7A)`，而 `case` 只支持精确匹配，需要将 1400 多个值逐个列出，代码过于庞大。
   ```scheme
   (define (char-upcase char)
     (unless (char? char)
       (error 'type-error "char-upcase: parameter must be character"))
     (let ((cp (char->integer char)))
       (cond
         ((<= #x61 cp #x7A) (integer->char (+ cp -32)))
         ...
         (else char))))
   ```
4. **移除 C 实现**：删除 `src/s7_scheme_char.c` 中的 `char_upcase_p_p`、`char_upcase_p_p_unchecked`、`g_char_upcase`、`g_char_downcase`，以及 `s7.c` 中的注册代码。
5. **更新测试**：确保现有测试用例覆盖 ASCII、Latin-1 和 Unicode 字符，并增加更多边界情况测试。

### 已知问题

`bin/gf fmt` 会破坏 Unicode 字符字面量（如 `#\à` 会被替换为 `#\�`）。因此 `tests/scheme/char/char-upcase-test.scm` 和 `tests/scheme/char/char-downcase-test.scm` 不应运行 `bin/gf fmt`。
